## 2. 二进制基础

### 2.1 为什么用二进制？

计算机用电信号表示数据：
- 高电平 = 1
- 低电平 = 0

只有两种状态，所以用二进制（0 和 1）。

### 2.2 进制转换

**十进制 → 二进制：**
```
13 ÷ 2 = 6 ... 1
 6 ÷ 2 = 3 ... 0
 3 ÷ 2 = 1 ... 1
 1 ÷ 2 = 0 ... 1

从下往上读：13 = 1101（二进制）
```

**二进制 → 十进制：**
```
1101 = 1×2³ + 1×2² + 0×2¹ + 1×2⁰
     = 8 + 4 + 0 + 1
     = 13
```

**Go 中的进制表示：**
```go
a := 13       // 十进制
b := 0b1101   // 二进制（0b 前缀）
c := 015      // 八进制（0 前缀）
d := 0xD      // 十六进制（0x 前缀）

fmt.Println(a, b, c, d)  // 都是 13
```

### 2.3 位（bit）和字节（byte）

```
1 bit  = 0 或 1（最小单位）
1 byte = 8 bits

1 KB = 1024 bytes
1 MB = 1024 KB
1 GB = 1024 MB
```

**Go 的数据类型大小：**
```go
int8   → 1 byte  → 8 bits
int16  → 2 bytes → 16 bits
int32  → 4 bytes → 32 bits
int64  → 8 bytes → 64 bits
```

## 3. 整数的表示

### 3.1 无符号整数（uint）

直接用二进制表示，没有负数。

```
uint8 的范围：0 ~ 255（2⁸ - 1）

0   = 00000000
1   = 00000001
127 = 01111111
255 = 11111111
```

**Go 中的无符号整数：**
```go
var a uint8 = 255
a = a + 1  // 溢出！变成 0
fmt.Println(a)  // 输出 0
```

### 3.2 有符号整数（int）- 补码表示

**问题：如何表示负数？**

答案：用**补码**（Two's Complement）

**补码规则：**
- 最高位是符号位：0 表示正数，1 表示负数
- 正数的补码 = 原码
- 负数的补码 = 取反 + 1

**int8 的例子：**
```
+5 的补码：
  5 = 00000101（正数，直接表示）

-5 的补码：
  5 = 00000101
取反 = 11111010
 +1 = 11111011  ← 这就是 -5 的补码
```

**int8 的范围：-128 ~ 127**
```
-128 = 10000000
-1   = 11111111
 0   = 00000000
 1   = 00000001
 127 = 01111111
```

### 3.3 为什么用补码？

**好处：加法和减法可以统一处理！**

```
5 + (-3) = 5 - 3 = 2

用补码计算：
  00000101  (5)
+ 11111101  (-3 的补码)
= 00000010  (2)

CPU 只需要一个加法器，不需要单独的减法器！
```

### 3.4 类型转换的坑

```go
var a int8 = -1
// -1 的补码：11111111

var b uint8 = uint8(a)
// 直接把 11111111 当作无符号数
// 11111111 = 255

fmt.Println(b)  // 输出 255
```

**记住：类型转换只是重新解释二进制，不会改变二进制本身！**

## 4. 浮点数的表示

### 4.1 IEEE 754 标准

浮点数用科学计数法表示：

```
12.5 = 1.25 × 10¹（十进制）
     = 1.1001 × 2³（二进制）
```

**float32 的结构（32 位）：**
```
┌───┬──────────┬───────────────────────┐
│ S │ Exponent │       Mantissa        │
│ 1 │    8     │          23           │ bits
└───┴──────────┴───────────────────────┘
  ↓       ↓              ↓
符号位   指数部分       尾数部分
```

**float64 的结构（64 位）：**
```
┌───┬──────────────┬────────────────────────────────────────────┐
│ S │   Exponent   │                  Mantissa                  │
│ 1 │      11      │                    52                      │ bits
└───┴──────────────┴────────────────────────────────────────────┘
```

### 4.2 精度问题

**为什么 0.1 + 0.2 ≠ 0.3？**

```go
fmt.Println(0.1 + 0.2)        // 0.30000000000000004
fmt.Println(0.1 + 0.2 == 0.3) // false
```

**原因：** 0.1 在二进制中是无限循环小数！

```
0.1（十进制）= 0.0001100110011...（二进制，无限循环）
```

就像 1/3 = 0.333... 在十进制中无法精确表示一样。

**解决方案：**
```go
// 方法1：使用误差范围比较
const epsilon = 1e-9
if math.Abs(a - b) < epsilon {
    // 认为相等
}

// 方法2：用整数计算（金额用分而不是元）
price := 199  // 199 分 = 1.99 元
```

### 4.3 特殊值

```go
var inf = math.Inf(1)   // 正无穷
var ninf = math.Inf(-1) // 负无穷
var nan = math.NaN()    // Not a Number

fmt.Println(1.0 / 0.0)  // +Inf
fmt.Println(-1.0 / 0.0) // -Inf
fmt.Println(0.0 / 0.0)  // NaN
```

## 5. 字符的表示

### 5.1 ASCII 码

最早的字符编码，用 7 位表示 128 个字符：

```
字符    ASCII码（十进制）   二进制
'A'     65                 01000001
'a'     97                 01100001
'0'     48                 00110000
' '     32                 00100000
'\n'    10                 00001010
```

**Go 中查看 ASCII：**
```go
fmt.Println(byte('A'))  // 65
fmt.Println(byte('a'))  // 97
fmt.Println('a' - 'A')  // 32（大小写差 32）
```

### 5.2 Unicode 和 UTF-8

**问题：** ASCII 只有 128 个字符，中文怎么办？

**解决：** Unicode 给全世界所有字符编号

```
'A' → U+0041
'中' → U+4E2D
'😀' → U+1F600
```

**UTF-8：** Unicode 的一种编码方式，变长编码

```
字符范围              UTF-8 字节数
U+0000 ~ U+007F      1 字节（兼容 ASCII）
U+0080 ~ U+07FF      2 字节
U+0800 ~ U+FFFF      3 字节（中文在这里）
U+10000 ~ U+10FFFF   4 字节（emoji 在这里）
```

**Go 中的字符串：**
```go
s := "你好"
fmt.Println(len(s))         // 6（字节数，每个中文 3 字节）
fmt.Println(len([]rune(s))) // 2（字符数）

// 遍历字符
for _, r := range s {
    fmt.Printf("%c ", r)  // 你 好
}
```

### 5.3 Go 的 rune 类型

```go
// byte = uint8，表示一个字节
// rune = int32，表示一个 Unicode 码点

var b byte = 'A'  // ASCII 字符用 byte
var r rune = '中' // 中文用 rune

fmt.Println(b)  // 65
fmt.Println(r)  // 20013（'中' 的 Unicode 码点）
```

## 6. 位运算

### 6.1 基本位运算

| 运算 | 符号 | 说明 | 例子 |
|------|------|------|------|
| 与 | `&` | 都是 1 才是 1 | `5 & 3 = 1` |
| 或 | `\|` | 有一个 1 就是 1 | `5 \| 3 = 7` |
| 异或 | `^` | 不同才是 1 | `5 ^ 3 = 6` |
| 取反 | `^` | 0 变 1，1 变 0 | `^5 = -6` |
| 左移 | `<<` | 左移 n 位 = ×2ⁿ | `5 << 2 = 20` |
| 右移 | `>>` | 右移 n 位 = ÷2ⁿ | `20 >> 2 = 5` |

### 6.2 位运算例子

```go
// 与运算：判断奇偶
n & 1 == 0  // 偶数
n & 1 == 1  // 奇数

// 左移：快速乘 2
n << 1  // n * 2
n << 3  // n * 8

// 右移：快速除 2
n >> 1  // n / 2

// 异或：交换两个数
a = a ^ b
b = a ^ b
a = a ^ b
// 现在 a 和 b 的值交换了
```

### 6.3 位运算实际应用

**权限管理：**
```go
const (
    Read    = 1 << 0  // 001 = 1
    Write   = 1 << 1  // 010 = 2
    Execute = 1 << 2  // 100 = 4
)

// 设置权限
permission := Read | Write  // 011 = 3

// 检查权限
hasRead := permission & Read != 0    // true
hasExecute := permission & Execute != 0  // false

// 添加权限
permission |= Execute  // 111 = 7

// 移除权限
permission &^= Write   // 101 = 5
```

## 7. 大端和小端

### 7.1 字节序

多字节数据在内存中的存储顺序：

```
数字 0x12345678（4 字节）

大端（Big Endian）：高位在前
地址：  0x00  0x01  0x02  0x03
内容：  0x12  0x34  0x56  0x78

小端（Little Endian）：低位在前
地址：  0x00  0x01  0x02  0x03
内容：  0x78  0x56  0x34  0x12
```

**x86/x64 CPU 用小端，网络传输用大端**

### 7.2 Go 中处理字节序

```go
import "encoding/binary"

num := uint32(0x12345678)
buf := make([]byte, 4)

// 大端
binary.BigEndian.PutUint32(buf, num)
fmt.Printf("%x\n", buf)  // [12 34 56 78]

// 小端
binary.LittleEndian.PutUint32(buf, num)
fmt.Printf("%x\n", buf)  // [78 56 34 12]
```

## 8. 练习题

### 练习1：进制转换

**Q1：把十进制 100 转换成二进制**

<details>
<summary>查看答案</summary>

```
100 ÷ 2 = 50 ... 0
 50 ÷ 2 = 25 ... 0
 25 ÷ 2 = 12 ... 1
 12 ÷ 2 = 6  ... 0
  6 ÷ 2 = 3  ... 0
  3 ÷ 2 = 1  ... 1
  1 ÷ 2 = 0  ... 1

100 = 1100100（二进制）
```

</details>

**Q2：二进制 10110 等于十进制多少？**

<details>
<summary>查看答案</summary>

```
10110 = 1×2⁴ + 0×2³ + 1×2² + 1×2¹ + 0×2⁰
      = 16 + 0 + 4 + 2 + 0
      = 22
```

</details>

### 练习2：补码计算

**Q：int8 类型，-5 的补码是多少？**

<details>
<summary>查看答案</summary>

```
5 的原码：00000101
取反：    11111010
加 1：    11111011

-5 的补码是 11111011
```

验证：
```go
var a int8 = -5
fmt.Printf("%08b\n", byte(a))  // 11111011
```

</details>

### 练习3：类型转换

**Q：以下代码输出什么？为什么？**
```go
var a int8 = -128
var b uint8 = uint8(a)
fmt.Println(b)
```

<details>
<summary>查看答案</summary>

输出：`128`

原因：
- -128 的补码是 `10000000`
- 转成 uint8 后，`10000000` 被解释为无符号数
- `10000000` = 128

</details>

### 练习4：字符串长度

**Q：以下代码输出什么？**
```go
s := "Go语言"
fmt.Println(len(s))
fmt.Println(len([]rune(s)))
```

<details>
<summary>查看答案</summary>

输出：
```
8
4
```

原因：
- `len(s)` 返回字节数
- "Go" = 2 字节，"语言" = 6 字节（每个中文 3 字节）
- 总共 8 字节

- `len([]rune(s))` 返回字符数
- "G", "o", "语", "言" = 4 个字符

</details>
