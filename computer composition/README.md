## 💡 为什么 Go 开发者要学这个？

**场景1：性能优化**
```go
// 为什么这样写更快？
for i := 0; i < len(arr); i++ {
    sum += arr[i]  // 顺序访问，Cache 友好
}

// 为什么这样写更慢？
for i := 0; i < len(arr); i += 64 {
    sum += arr[i]  // 跳跃访问，Cache 不友好
}
```
学完存储系统你就懂了！

**场景2：理解数据类型**
```go
var a int32 = -1
var b uint32 = uint32(a)
fmt.Println(b)  // 为什么是 4294967295？
```
学完数据表示你就懂了！

**场景3：理解并发**
```go
// 为什么需要原子操作？
atomic.AddInt64(&counter, 1)
```
学完 CPU 原理你就懂了！

## 🎯 学习目标

学完这套教程，你应该能回答：
1. 程序是如何在计算机上运行的？
2. 为什么 int64 比 int32 占用更多内存？
3. 为什么顺序访问数组比随机访问快？
4. CPU 是如何执行 Go 代码的？
5. 什么是 Cache？为什么它这么重要？
