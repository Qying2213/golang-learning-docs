# 计算机系统概述 ⭐⭐⭐

> 本章目标：理解计算机的基本组成和工作原理，知道程序是如何运行的

---

## 1. 计算机能做什么？

计算机本质上只会做一件事：**按照指令处理数据**

```
输入数据 → 处理数据 → 输出结果

例子：
1 + 2 → CPU 计算 → 3
```

你写的 Go 代码：
```go
a := 1
b := 2
c := a + b
```

最终会变成 CPU 能理解的指令，让 CPU 去执行。

## 2. 冯诺依曼架构

### 2.1 核心思想

1946 年，冯诺依曼提出了计算机的基本架构，至今仍在使用：

**核心思想：程序和数据都存储在内存中**

```
┌─────────────────────────────────────────────────────────────┐
│                        计算机系统                            │
│                                                              │
│  ┌─────────────┐      ┌─────────────┐      ┌─────────────┐ │
│  │   输入设备   │      │     CPU     │      │   输出设备   │ │
│  │  (键盘鼠标)  │      │  (处理器)   │      │  (显示器)   │ │
│  └──────┬──────┘      └──────┬──────┘      └──────┬──────┘ │
│         │                    │                    │         │
│         │         ┌──────────┴──────────┐        │         │
│         │         │                     │        │         │
│         └────────→│       内存          │←───────┘         │
│                   │  (存储程序和数据)    │                   │
│                   └─────────────────────┘                   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 五大部件

| 部件 | 功能 | 类比 |
|------|------|------|
| **运算器** | 执行算术和逻辑运算 | 计算器 |
| **控制器** | 控制程序执行流程 | 指挥官 |
| **存储器** | 存储程序和数据 | 仓库 |
| **输入设备** | 接收外部数据 | 耳朵、眼睛 |
| **输出设备** | 输出处理结果 | 嘴巴、手 |

**运算器 + 控制器 = CPU（中央处理器）**

### 2.3 工作流程

```
1. 程序和数据从硬盘加载到内存
2. CPU 从内存读取指令
3. CPU 解析指令，知道要做什么
4. CPU 执行指令（可能需要读写内存）
5. 重复 2-4，直到程序结束
```

**Go 程序的运行过程：**
```
go run main.go
      ↓
编译成机器码（二进制指令）
      ↓
加载到内存
      ↓
CPU 逐条执行指令
      ↓
程序运行完毕
```

## 3. 计算机的层次结构

### 3.1 从硬件到软件

```
┌─────────────────────────────────────┐
│           应用程序层                 │  ← 你写的 Go 程序
│         (Go、Python、Java)          │
├─────────────────────────────────────┤
│           操作系统层                 │  ← Linux、Windows、macOS
│      (进程管理、内存管理、文件系统)   │
├─────────────────────────────────────┤
│           指令集架构层               │  ← CPU 能理解的指令
│         (x86、ARM、RISC-V)          │
├─────────────────────────────────────┤
│           微架构层                   │  ← CPU 内部实现
│        (流水线、缓存、分支预测)       │
├─────────────────────────────────────┤
│           硬件层                     │  ← 物理电路
│         (晶体管、电路、芯片)          │
└─────────────────────────────────────┘
```

### 3.2 你需要了解哪些层？

作为 Go 后端开发者：

| 层次 | 需要了解程度 | 原因 |
|------|-------------|------|
| 应用程序层 | 精通 | 这是你的主战场 |
| 操作系统层 | 熟悉 | 理解进程、内存、I/O |
| 指令集架构层 | 了解 | 理解性能优化 |
| 微架构层 | 了解 | 理解 Cache、并发 |
| 硬件层 | 知道即可 | 一般不需要深入 |

## 4. 程序是如何运行的？

### 4.1 从源代码到执行

```
┌──────────────────────────────────────────────────────────────┐
│                    Go 程序的生命周期                          │
│                                                               │
│   main.go          编译器           可执行文件      CPU       │
│  ┌────────┐      ┌────────┐       ┌────────┐    ┌────────┐  │
│  │ func   │      │        │       │ 二进制  │    │        │  │
│  │ main() │  →   │ go build│  →   │  指令   │ →  │  执行  │  │
│  │ {...}  │      │        │       │        │    │        │  │
│  └────────┘      └────────┘       └────────┘    └────────┘  │
│                                                               │
│   源代码            编译             机器码         运行       │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 实际例子

```go
// main.go
package main

func main() {
    a := 1
    b := 2
    c := a + b
    println(c)
}
```

**编译后的汇编代码（简化版）：**
```assembly
; a := 1
MOV  $1, AX        ; 把 1 放到 AX 寄存器

; b := 2
MOV  $2, BX        ; 把 2 放到 BX 寄存器

; c := a + b
ADD  BX, AX        ; AX = AX + BX，结果在 AX 中

; println(c)
CALL println       ; 调用打印函数
```

**查看 Go 程序的汇编：**
```bash
go build -gcflags="-S" main.go
```

### 4.3 指令执行周期

CPU 执行每条指令都经过这些步骤：

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  取指   │ →  │  译码   │ →  │  执行   │ →  │  访存   │ →  │  写回   │
│ Fetch   │    │ Decode  │    │ Execute │    │ Memory  │    │ Write   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     ↓              ↓              ↓              ↓              ↓
从内存取出      解析指令       执行运算       读写内存       保存结果
  指令        知道做什么      （如加法）     （如果需要）    到寄存器
```

## 5. 计算机的性能指标

### 5.1 主要指标

| 指标 | 含义 | 单位 | 例子 |
|------|------|------|------|
| **主频** | CPU 每秒时钟周期数 | Hz | 3.5 GHz = 每秒 35 亿个周期 |
| **CPI** | 每条指令需要的周期数 | 周期/指令 | CPI = 2 表示平均每条指令需要 2 个周期 |
| **MIPS** | 每秒执行百万条指令 | 百万条/秒 | 1000 MIPS = 每秒 10 亿条指令 |
| **内存容量** | 内存大小 | GB | 16 GB |
| **内存带宽** | 内存传输速度 | GB/s | 50 GB/s |

### 5.2 性能计算

```
CPU 执行时间 = 指令数 × CPI × 时钟周期

例子：
- 程序有 1000 万条指令
- CPI = 2
- 主频 = 2 GHz（时钟周期 = 0.5 ns）

执行时间 = 10,000,000 × 2 × 0.5 ns = 10 ms
```

### 5.3 影响 Go 程序性能的因素

```go
// 因素1：指令数量
// 循环 1000 次 vs 循环 100 次，指令数差 10 倍

// 因素2：内存访问
// 顺序访问 vs 随机访问，速度差很多倍

// 因素3：分支预测
// if-else 预测错误会浪费 CPU 周期
```

## 6. 现代计算机架构

### 6.1 多核 CPU

```
┌─────────────────────────────────────────────┐
│                   CPU                        │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐     │
│  │  Core 0 │  │  Core 1 │  │  Core 2 │ ... │
│  │  核心0  │  │  核心1  │  │  核心2  │     │
│  └─────────┘  └─────────┘  └─────────┘     │
│        ↓           ↓           ↓            │
│  ┌─────────────────────────────────────┐   │
│  │           共享 L3 Cache              │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
                     ↓
              ┌─────────────┐
              │    内存     │
              └─────────────┘
```

**Go 的 goroutine 可以在多个核心上并行执行！**

```go
// GOMAXPROCS 设置使用的核心数
runtime.GOMAXPROCS(4)  // 使用 4 个核心
```

### 6.2 存储层次

```
速度快 ←─────────────────────────────────→ 速度慢
容量小 ←─────────────────────────────────→ 容量大
价格贵 ←─────────────────────────────────→ 价格便宜

┌─────────┐
│ 寄存器  │  ← 最快，几十个，CPU 内部
├─────────┤
│ L1 Cache│  ← 很快，32-64 KB，每个核心独有
├─────────┤
│ L2 Cache│  ← 快，256 KB-1 MB，每个核心独有
├─────────┤
│ L3 Cache│  ← 较快，几 MB-几十 MB，多核共享
├─────────┤
│  内存   │  ← 中等，几 GB-几百 GB
├─────────┤
│  SSD    │  ← 较慢，几百 GB-几 TB
├─────────┤
│  HDD    │  ← 慢，几 TB
└─────────┘
```

**访问延迟对比：**
| 存储 | 延迟 | 类比 |
|------|------|------|
| 寄存器 | 0.5 ns | 眨眼 |
| L1 Cache | 1 ns | 心跳一次 |
| L2 Cache | 4 ns | 打个喷嚏 |
| L3 Cache | 12 ns | 深呼吸 |
| 内存 | 100 ns | 泡杯咖啡 |
| SSD | 100,000 ns | 看完一集电视剧 |
| HDD | 10,000,000 ns | 睡一觉 |

## 7. 练习题

### 练习1：概念理解

**Q1：冯诺依曼架构的核心思想是什么？**

<details>
<summary>查看答案</summary>

**存储程序**：程序和数据都存储在内存中，CPU 从内存中读取指令并执行。

这意味着：
1. 程序可以被修改（因为存在内存中）
2. 程序和数据用同样的方式存储
3. CPU 按顺序执行指令（除非遇到跳转）

</details>

**Q2：CPU 的五大部件是什么？**

<details>
<summary>查看答案</summary>

1. **运算器**：执行算术和逻辑运算
2. **控制器**：控制指令执行流程
3. **存储器**：存储程序和数据（内存）
4. **输入设备**：键盘、鼠标等
5. **输出设备**：显示器、打印机等

其中，运算器 + 控制器 = CPU

</details>

**Q3：为什么需要多级缓存（L1、L2、L3）？**

<details>
<summary>查看答案</summary>

因为 CPU 和内存的速度差距太大：
- CPU 执行一条指令：约 0.5 ns
- 访问内存：约 100 ns

如果每次都访问内存，CPU 大部分时间在等待。

多级缓存的作用：
- L1 最快但最小，存放最常用的数据
- L2 较快较大，存放次常用的数据
- L3 更大，多核共享

这样大部分数据访问都能在缓存中完成，不用等内存。

</details>

### 练习2：计算题

**Q：一个程序有 1 亿条指令，CPI = 1.5，CPU 主频 = 3 GHz，执行时间是多少？**

<details>
<summary>查看答案</summary>

```
执行时间 = 指令数 × CPI ÷ 主频
        = 100,000,000 × 1.5 ÷ 3,000,000,000
        = 150,000,000 ÷ 3,000,000,000
        = 0.05 秒
        = 50 毫秒
```

</details>

### 练习3：Go 相关

**Q：为什么 Go 程序需要编译？直接运行源代码不行吗？**

<details>
<summary>查看答案</summary>

CPU 只能执行机器码（二进制指令），不能直接理解 Go 源代码。

编译的作用：
1. 把 Go 代码翻译成机器码
2. 进行优化（让程序更快）
3. 检查语法错误

```
main.go  →  go build  →  main（可执行文件）→  CPU 执行
 源代码       编译          机器码
```

Python 等解释型语言看起来不用编译，但实际上是解释器在运行时逐行翻译执行，所以比编译型语言慢。

</details>

## 8. 本章小结

```
计算机组成：
├── 冯诺依曼架构
│   ├── 存储程序思想
│   └── 五大部件：运算器、控制器、存储器、输入、输出
│
├── 程序运行过程
│   ├── 源代码 → 编译 → 机器码 → CPU 执行
│   └── 取指 → 译码 → 执行 → 访存 → 写回
│
├── 存储层次
│   └── 寄存器 → L1 → L2 → L3 → 内存 → SSD → HDD
│
└── 现代架构
    ├── 多核 CPU
    └── 缓存系统
```

**关键理解：**
1. 程序最终都是 CPU 执行的机器指令
2. 内存访问比 CPU 计算慢很多，所以需要缓存
3. 多核 CPU 让并行成为可能（Go 的 goroutine）
