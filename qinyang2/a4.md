## 五、面试技巧

### 回答问题的 STAR 法则

```
S - Situation（情境）
    描述问题的背景

T - Task（任务）
    你需要完成什么

A - Action（行动）
    你采取了什么行动

R - Result（结果）
    取得了什么成果
```

### 项目介绍模板

```
1. 项目背景
   - 业务场景
   - 技术挑战

2. 技术方案
   - 架构设计
   - 技术选型
   - 核心实现

3. 个人贡献
   - 负责的模块
   - 解决的问题

4. 项目成果
   - 量化指标
   - 业务价值
```

### 常见问题回答

```
"你有什么问题想问我？"
├── 团队的技术栈是什么？
├── 这个岗位的主要职责？
├── 团队的技术氛围如何？
├── 公司的技术发展方向？
└── 入职后的培养计划？

"你的优点和缺点？"
├── 优点：结合具体例子
├── 缺点：说一个真实但不致命的缺点
└── 强调正在改进

"为什么离开上家公司？"
├── 寻求更大的发展空间
├── 想接触更有挑战的项目
├── 技术方向的调整
└── 不要说前公司坏话

"期望薪资是多少？"
├── 了解市场行情
├── 给一个范围
├── 强调更看重发展
└── 可以谈
```

## 七、薪资谈判

```
准备工作：
├── 了解市场行情

├── 明确自己的底线

谈判技巧：
├── 不要先报价
├── 给范围而不是具体数字
├── 强调自己的价值
```
