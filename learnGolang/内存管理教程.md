# Go 内存管理教程

> 目标：理解栈、堆、逃逸分析，知道变量分配在哪里

---

## 1. 栈和堆是什么

程序运行时，内存分为两个区域：

```
┌─────────────────────────────────────┐
│              栈 (Stack)              │  ← 小、快、自动管理
│  - 函数的局部变量                    │
│  - 函数调用信息                      │
│  - 自动分配和释放                    │
├─────────────────────────────────────┤
│              堆 (Heap)               │  ← 大、慢、需要 GC
│  - 动态分配的内存                    │
│  - 需要垃圾回收 (GC)                 │
│  - 生命周期不确定的变量              │
└─────────────────────────────────────┘
```

### 1.1 栈 (Stack)

- **特点**：快、自动管理
- **分配**：函数调用时自动分配
- **释放**：函数返回时自动释放
- **大小**：Go 的 goroutine 栈初始 2KB，可动态增长

```go
func foo() {
    x := 10      // x 在栈上
    y := "hello" // y 在栈上（如果没逃逸）
}  // 函数结束，x 和 y 自动释放
```

### 1.2 堆 (Heap)

- **特点**：慢、需要 GC
- **分配**：运行时动态分配
- **释放**：垃圾回收器 (GC) 负责
- **大小**：受系统内存限制

```go
func foo() *int {
    x := 10
    return &x  // x 逃逸到堆上，因为函数外还要用
}
```

## 2. 逃逸分析 (Escape Analysis)

**逃逸分析**：编译器决定变量分配在栈上还是堆上。

### 2.1 什么是逃逸

变量"逃出"了函数的作用域，就叫逃逸。

```go
// 不逃逸：x 只在函数内使用
func noEscape() int {
    x := 10
    return x  // 返回值，不是指针
}

// 逃逸：x 的指针被返回，函数外还要用
func escape() *int {
    x := 10
    return &x  // 返回指针，x 逃逸到堆
}
```

### 2.2 查看逃逸分析

```bash
go build -gcflags="-m" main.go
```

输出示例：
```
./main.go:5:2: moved to heap: x
./main.go:10:2: x does not escape
```

### 2.3 常见逃逸场景

**场景 1：返回局部变量的指针**
```go
func escape1() *int {
    x := 10
    return &x  // 逃逸！
}
```

**场景 2：发送指针到 channel**
```go
func escape2() {
    ch := make(chan *int)
    x := 10
    ch <- &x  // 逃逸！
}
```

**场景 3：闭包引用局部变量**
```go
func escape3() func() int {
    x := 10
    return func() int {
        return x  // x 被闭包引用，逃逸！
    }
}
```

**场景 4：interface{} 类型**
```go
func escape4() {
    x := 10
    fmt.Println(x)  // x 逃逸！因为 Println 参数是 interface{}
}
```

**场景 5：slice/map 太大**
```go
func escape5() {
    s := make([]int, 10000)  // 太大，逃逸到堆
}
```

**场景 6：slice append 可能扩容**
```go
func escape6() []int {
    s := make([]int, 0)
    s = append(s, 1)  // 可能扩容，逃逸
    return s
}
```

## 3. 栈和堆的性能差异

```go
// 栈分配：快
func stackAlloc() {
    x := 10  // 栈上，几个 CPU 指令
    _ = x
}

// 堆分配：慢
func heapAlloc() *int {
    x := new(int)  // 堆上，需要调用内存分配器
    return x
}
```

**为什么堆慢**：
1. 需要调用内存分配器
2. 需要 GC 回收
3. 可能触发 GC 暂停

**基准测试**：
```go
func BenchmarkStack(b *testing.B) {
    for i := 0; i < b.N; i++ {
        x := 10
        _ = x
    }
}

func BenchmarkHeap(b *testing.B) {
    for i := 0; i < b.N; i++ {
        x := new(int)
        _ = x
    }
}

// 结果：栈分配比堆分配快 10-100 倍
```

## 4. Go 内存分配器

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        Go Runtime                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   ┌─────────┐  ┌─────────┐  ┌─────────┐                     │
│   │  mcache │  │  mcache │  │  mcache │   ← 每个 P 一个     │
│   │  (P0)   │  │  (P1)   │  │  (P2)   │      无锁分配       │
│   └────┬────┘  └────┬────┘  └────┬────┘                     │
│        │            │            │                           │
│        └────────────┼────────────┘                           │
│                     ↓                                        │
│              ┌──────────────┐                                │
│              │    mcentral   │  ← 所有 P 共享，有锁          │
│              └──────┬───────┘                                │
│                     ↓                                        │
│              ┌──────────────┐                                │
│              │    mheap      │  ← 管理整个堆内存             │
│              └──────────────┘                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 三级缓存

**mcache（线程缓存）**
- 每个 P 有一个
- 无锁分配，最快
- 存放小对象（<32KB）

**mcentral（中心缓存）**
- 所有 P 共享
- 有锁，较慢
- 给 mcache 补充内存

**mheap（堆）**
- 管理整个堆内存
- 向操作系统申请内存

### 4.3 对象大小分类

```
┌─────────────────────────────────────────────────────────────┐
│  大小分类          │  分配方式                              │
├───────────────────┼─────────────────────────────────────────┤
│  微对象 (<16B)    │  mcache.tiny，多个对象合并分配          │
│  小对象 (16B-32KB)│  mcache → mcentral → mheap             │
│  大对象 (>32KB)   │  直接从 mheap 分配                      │
└─────────────────────────────────────────────────────────────┘
```

## 5. 垃圾回收 (GC)

### 5.1 三色标记法

Go 使用**三色标记清除**算法：

```
白色：未访问的对象（可能是垃圾）
灰色：已访问但子对象未访问
黑色：已访问且子对象已访问（存活）

步骤：
1. 所有对象标记为白色
2. 从根对象开始，标记为灰色
3. 遍历灰色对象，把它引用的对象标记为灰色，自己变黑色
4. 重复直到没有灰色对象
5. 白色对象就是垃圾，回收
```

### 5.2 GC 触发条件

```go
// 1. 堆内存达到阈值（默认 100% 增长）
// 2. 手动触发
runtime.GC()

// 3. 定时触发（2 分钟没有 GC）
```

### 5.3 查看 GC 信息

```bash
GODEBUG=gctrace=1 go run main.go
```

输出示例：
```
gc 1 @0.012s 2%: 0.026+0.44+0.003 ms clock, 0.10+0.32/0.27/0+0.012 ms cpu, 4->4->0 MB, 5 MB goal, 4 P
```

### 5.4 减少 GC 压力

```go
// 1. 减少堆分配
// ❌ 每次循环都分配
for i := 0; i < 1000; i++ {
    s := make([]int, 100)
    process(s)
}

// ✅ 复用
s := make([]int, 100)
for i := 0; i < 1000; i++ {
    process(s)
}

// 2. 使用 sync.Pool 复用对象
var pool = sync.Pool{
    New: func() interface{} {
        return make([]byte, 1024)
    },
}

buf := pool.Get().([]byte)
// 使用 buf
pool.Put(buf)  // 用完放回去

// 3. 预分配
// ❌ 多次扩容
var s []int
for i := 0; i < 1000; i++ {
    s = append(s, i)
}

// ✅ 预分配
s := make([]int, 0, 1000)
for i := 0; i < 1000; i++ {
    s = append(s, i)
}
```

## 6. 实战：分析内存分配

### 6.1 逃逸分析

```go
package main

func main() {
    x := createInt()
    println(*x)
}

func createInt() *int {
    n := 42
    return &n
}
```

```bash
go build -gcflags="-m" main.go
# 输出：moved to heap: n
```

### 6.2 内存分析

```go
package main

import (
    "runtime"
    "fmt"
)

func main() {
    var m runtime.MemStats
    
    // 分配前
    runtime.ReadMemStats(&m)
    fmt.Printf("分配前 HeapAlloc: %d KB\n", m.HeapAlloc/1024)
    
    // 分配 100 万个 int
    s := make([]int, 1000000)
    _ = s
    
    // 分配后
    runtime.ReadMemStats(&m)
    fmt.Printf("分配后 HeapAlloc: %d KB\n", m.HeapAlloc/1024)
    fmt.Printf("GC 次数: %d\n", m.NumGC)
}
```

### 6.3 pprof 分析

```go
import _ "net/http/pprof"

func main() {

        http.ListenAndServe(":6060", nil)

    // ...

```


## 7. 常见问题

### 7.1 new 和 make 的区别

```go
// new：分配内存，返回指针，零值初始化
p := new(int)     // *int，值为 0
s := new([]int)   // *[]int，值为 nil

// make：只用于 slice/map/channel，返回值（不是指针）
s := make([]int, 10)     // []int，长度 10
m := make(map[string]int) // map[string]int
ch := make(chan int)      // chan int
```

### 7.2 值传递 vs 指针传递

```go
// 值传递：复制整个结构体
func byValue(u User) {
    // u 是副本
}

// 指针传递：只复制指针（8 字节）
func byPointer(u *User) {
    // u 指向原对象
}

// 建议：
// - 小结构体（<64 字节）：值传递
// - 大结构体：指针传递
// - 需要修改原对象：指针传递
```

### 7.3 内存泄漏场景

```go
// 1. goroutine 泄漏
func leak1() {
    ch := make(chan int)
    go func() {
        <-ch  // 永远阻塞，goroutine 泄漏
    }()
}

// 2. slice 引用大数组
func leak2() []int {
    big := make([]int, 1000000)
    return big[:10]  // 小 slice 引用大数组，大数组无法回收
}

// 修复：复制出来
func noLeak2() []int {
    big := make([]int, 1000000)
    small := make([]int, 10)
    copy(small, big[:10])
    return small
}

// 3. time.Ticker 没有 Stop
func leak3() {
    ticker := time.NewTicker(time.Second)
    // 忘记 ticker.Stop()
}
```

## 8. 总结

| 概念 | 说明 |
|------|------|
| 栈 | 快、自动管理、函数局部变量 |
| 堆 | 慢、需要 GC、逃逸的变量 |
| 逃逸分析 | 编译器决定变量在栈还是堆 |
| GC | 三色标记清除，回收堆上的垃圾 |
| mcache | 每个 P 的本地缓存，无锁 |
| mcentral | 中心缓存，有锁 |
| mheap | 管理整个堆 |

**优化建议**：
1. 减少逃逸（避免返回局部变量指针）
2. 预分配（slice、map）
3. 复用对象（sync.Pool）
4. 小对象用值传递

**查看工具**：
```bash
go build -gcflags="-m"     # 逃逸分析
GODEBUG=gctrace=1          # GC 追踪
go tool pprof              # 内存分析
```
