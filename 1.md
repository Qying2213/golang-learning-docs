## 二、季度规划

### Q1（1-3月）：Go 语言基础

**目标：** 把 Go 语言吃透，写代码不再查文档

| 月份 | 重点内容 | 产出 |
|------|----------|------|
| 1月 | 基础语法 + slice/map 原理 | 能解释 slice 扩容机制 |
| 2月 | interface + 并发编程 | 能写 goroutine + channel |
| 3月 | context + 错误处理 + 单元测试 | 能写规范的测试代码 |

**每月检验标准：**
- 1月：手写一个并发安全的 Map
- 2月：实现一个 Worker Pool
- 3月：为自己写的代码补充单元测试

---

### Q2（4-6月）：Web 开发

**目标：** 能独立完成后端接口开发

| 月份 | 重点内容 | 产出 |
|------|----------|------|
| 4月 | HTTP 协议 + Gin 框架基础 | 能写 CRUD 接口 |
| 5月 | 中间件 + JWT 认证 + 参数校验 | 能实现登录注册 |
| 6月 | 项目结构 + 错误处理 + 日志 | 搭建规范的项目骨架 |

**每月检验标准：**
- 4月：用 Gin 实现用户 CRUD 接口
- 5月：实现完整的登录注册（含 JWT）
- 6月：重构代码，形成规范的分层结构

---

### Q3（7-9月）：数据库

**目标：** 熟练使用 MySQL + Redis

| 月份 | 重点内容 | 产出 |
|------|----------|------|
| 7月 | MySQL 基础 + SQL 编写 + GORM | 能写复杂 SQL |
| 8月 | 索引原理 + 事务 + 慢查询优化 | 能分析 EXPLAIN |
| 9月 | Redis 数据类型 + 缓存设计 | 能用 Redis 做缓存 |

**每月检验标准：**
- 7月：设计一个电商系统的表结构
- 8月：优化一个慢查询（有 EXPLAIN 分析过程）
- 9月：实现文章阅读量统计 + 排行榜

---

### Q4（10-12月）：项目实战 + 面试准备

**目标：** 完成项目，准备跳槽/转正

| 月份 | 重点内容 | 产出 |
|------|----------|------|
| 10月 | 项目一：用户管理系统 | 完整可运行的项目 |
| 11月 | 项目二：博客系统 | 完整可运行的项目 |
| 12月 | 复习 + 面试准备 + 投简历 | 拿到 offer |

**每月检验标准：**
- 10月：用户系统上线到自己的服务器
- 11月：博客系统上线，能公开访问
- 12月：拿到至少 2 个面试机会

## 五、项目要求

### 项目一：用户管理系统（10月）

```
功能要求：
├── 用户注册（参数校验）
├── 用户登录（JWT）
├── 获取用户信息
├── 修改用户信息
├── 修改密码
├── 用户列表（分页）
└── 头像上传

技术要求：
├── Gin 框架
├── GORM + MySQL
├── Redis 缓存用户信息
├── JWT 认证中间件
├── 统一响应格式
├── 日志记录
└── 分层架构（handler/service/dao）
```

### 项目二：博客系统（11月）

```
功能要求：
├── 文章 CRUD
├── 文章分类
├── 文章标签（多对多）
├── 文章评论
├── 文章点赞
├── 阅读量统计
└── 热门文章排行

技术要求：
├── 包含项目一所有技术
├── 复杂 SQL（多表关联）
├── Redis 计数器（阅读量）
├── Redis ZSet（排行榜）
└── 简单的接口限流
```

## 七、避坑指南

### 不要做的事

```
✗ 不要同时学太多东西（专注当月主题）
✗ 不要只看不练（每个知识点都要写代码）
✗ 不要追求完美（先完成再完善）
✗ 不要忽视工作（工作中的代码是最好的练习）
✗ 不要熬夜学习（保持可持续性）
✗ 不要现在就学 K8s、微服务、分布式（基础不牢）
```

### 要做的事

```
✓ 每天写代码，哪怕只有 30 分钟
✓ 遇到问题先自己查，查不到再问
✓ 把工作中的代码当作学习材料
✓ 定期复习，防止遗忘
✓ 记录学习笔记（可以用 Notion 或 语雀）
✓ 每月检验自己是否达到目标
```
