## 1. 常见 Web 攻击 ⭐⭐⭐

### 1.1 SQL 注入

```
原理：用户输入被当作 SQL 代码执行

示例：
登录表单：用户名、密码

正常 SQL：
SELECT * FROM users WHERE username='admin' AND password='123456'

恶意输入：
用户名：admin' --
密码：随便

生成的 SQL：
SELECT * FROM users WHERE username='admin' --' AND password='随便'
                                           ↑
                                    注释掉了密码验证！

结果：不需要密码就能登录！
```

### 1.2 SQL 注入防御

```go
// ❌ 错误：字符串拼接
query := "SELECT * FROM users WHERE username='" + username + "'"
db.Query(query)

// ✅ 正确：参数化查询
db.Query("SELECT * FROM users WHERE username = ?", username)

// ✅ 正确：使用 ORM
db.Where("username = ?", username).First(&user)

// ✅ 正确：预编译语句
stmt, _ := db.Prepare("SELECT * FROM users WHERE username = ?")
stmt.Query(username)
```

### 1.3 XSS 跨站脚本攻击

```
原理：恶意脚本被注入到网页中执行

类型：
1. 存储型 XSS：恶意脚本存入数据库
2. 反射型 XSS：恶意脚本在 URL 中
3. DOM 型 XSS：前端 JS 处理不当

示例：
评论内容：<script>alert('XSS')</script>

如果直接显示，脚本会执行！

危害：
- 窃取 Cookie
- 劫持用户会话
- 钓鱼攻击
- 传播蠕虫
```

### 1.4 XSS 防御

```go
// 1. 输出转义
import "html"

// 转义 HTML 特殊字符
safeContent := html.EscapeString(userInput)
// <script> → &lt;script&gt;

// 2. 使用模板引擎（自动转义）
// Go html/template 默认转义
tmpl.Execute(w, data)

// 3. 设置 CSP 头
w.Header().Set("Content-Security-Policy", "default-src 'self'")

// 4. 设置 HttpOnly Cookie（防止 JS 读取）
http.SetCookie(w, &http.Cookie{
    Name:     "session",
    Value:    sessionID,
    HttpOnly: true,
    Secure:   true,
})
```

### 1.5 CSRF 跨站请求伪造

```
原理：利用用户已登录的身份，发送恶意请求

场景：
1. 用户登录了银行网站 bank.com
2. 用户访问恶意网站 evil.com
3. evil.com 页面包含：
   <img src="https://bank.com/transfer?to=hacker&amount=10000">
4. 浏览器自动带上 bank.com 的 Cookie
5. 转账请求被执行！

危害：
- 以用户身份执行操作
- 修改用户数据
- 发送恶意请求
```

### 1.6 CSRF 防御

```go
// 1. CSRF Token
// 生成 Token
token := generateCSRFToken()
session.Set("csrf_token", token)

// 表单中包含 Token
<input type="hidden" name="csrf_token" value="{{.CSRFToken}}">

// 验证 Token
func CSRFMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        if r.Method == "POST" {
            sessionToken := session.Get("csrf_token")
            formToken := r.FormValue("csrf_token")
            if sessionToken != formToken {
                http.Error(w, "CSRF token invalid", 403)
                return
            }
        }
        next.ServeHTTP(w, r)
    })
}

// 2. SameSite Cookie
http.SetCookie(w, &http.Cookie{
    Name:     "session",
    Value:    sessionID,
    SameSite: http.SameSiteStrictMode,  // 严格模式
})

// 3. 验证 Referer/Origin 头
origin := r.Header.Get("Origin")
if origin != "https://mysite.com" {
    http.Error(w, "Invalid origin", 403)
    return
}
```

## 2. 其他常见攻击 ⭐⭐

### 2.1 点击劫持

```
原理：用透明 iframe 覆盖正常页面

防御：
// 设置 X-Frame-Options
w.Header().Set("X-Frame-Options", "DENY")
// 或
w.Header().Set("X-Frame-Options", "SAMEORIGIN")

// CSP frame-ancestors
w.Header().Set("Content-Security-Policy", "frame-ancestors 'none'")
```

### 2.2 文件上传漏洞

```go
// 防御措施

// 1. 验证文件类型
allowedTypes := map[string]bool{
    "image/jpeg": true,
    "image/png":  true,
    "image/gif":  true,
}

file, header, _ := r.FormFile("file")
contentType := header.Header.Get("Content-Type")
if !allowedTypes[contentType] {
    http.Error(w, "Invalid file type", 400)
    return
}

// 2. 检查文件扩展名
ext := filepath.Ext(header.Filename)
if ext != ".jpg" && ext != ".png" && ext != ".gif" {
    http.Error(w, "Invalid file extension", 400)
    return
}

// 3. 限制文件大小
r.Body = http.MaxBytesReader(w, r.Body, 10<<20) // 10MB

// 4. 重命名文件
newName := uuid.New().String() + ext

// 5. 存储到非 Web 目录
savePath := "/data/uploads/" + newName
```

### 2.3 敏感信息泄露

```go
// 1. 不要在日志中记录敏感信息
// ❌ 错误
log.Printf("User login: %s, password: %s", username, password)
// ✅ 正确
log.Printf("User login: %s", username)

// 2. 错误信息不要暴露细节
// ❌ 错误
c.JSON(500, gin.H{"error": err.Error()})
// ✅ 正确
log.Printf("Error: %v", err)  // 记录到日志
c.JSON(500, gin.H{"error": "Internal server error"})

// 3. 生产环境关闭调试模式
gin.SetMode(gin.ReleaseMode)

// 4. 敏感配置使用环境变量
dbPassword := os.Getenv("DB_PASSWORD")
```

## 3. JWT 认证 ⭐⭐⭐

### 3.1 JWT 结构

```
JWT = JSON Web Token

结构：Header.Payload.Signature

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.
eyJ1c2VyX2lkIjoxLCJleHAiOjE3MDk0NTYwMDB9.
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c

Header（头部）：
{
    "alg": "HS256",  // 签名算法
    "typ": "JWT"     // 类型
}

Payload（载荷）：
{
    "user_id": 1,
    "username": "admin",
    "exp": 1709456000,  // 过期时间
    "iat": 1709369600   // 签发时间
}

Signature（签名）：
HMACSHA256(
    base64UrlEncode(header) + "." + base64UrlEncode(payload),
    secret
)
```

### 3.2 JWT 工作流程

```
1. 用户登录
   POST /login
   {username, password}

2. 服务器验证，生成 JWT
   → 返回 {token: "eyJ..."}

3. 客户端保存 Token
   localStorage 或 Cookie

4. 后续请求携带 Token
   Authorization: Bearer eyJ...

5. 服务器验证 Token
   - 验证签名
   - 检查过期时间
   - 提取用户信息
```

### 3.3 Go 实现 JWT

```go
package main

import (
    "time"
    "github.com/golang-jwt/jwt/v5"
)

var jwtSecret = []byte("your-secret-key")

// 自定义 Claims
type Claims struct {
    UserID   int64  `json:"user_id"`
    Username string `json:"username"`
    jwt.RegisteredClaims
}

// 生成 Token
func GenerateToken(userID int64, username string) (string, error) {
    claims := Claims{
        UserID:   userID,
        Username: username,
        RegisteredClaims: jwt.RegisteredClaims{
            ExpiresAt: jwt.NewNumericDate(time.Now().Add(24 * time.Hour)),
            IssuedAt:  jwt.NewNumericDate(time.Now()),
            Issuer:    "myapp",
        },
    }

    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString(jwtSecret)
}

// 解析 Token
func ParseToken(tokenString string) (*Claims, error) {
    token, err := jwt.ParseWithClaims(tokenString, &Claims{}, func(token *jwt.Token) (interface{}, error) {
        return jwtSecret, nil
    })

    if err != nil {
        return nil, err
    }

    if claims, ok := token.Claims.(*Claims); ok && token.Valid {
        return claims, nil
    }

    return nil, jwt.ErrSignatureInvalid
}
```

### 3.4 JWT 中间件

```go
func JWTMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        // 获取 Token
        authHeader := c.GetHeader("Authorization")
        if authHeader == "" {
            c.JSON(401, gin.H{"error": "Missing authorization header"})
            c.Abort()
            return
        }

        // 解析 Bearer Token
        parts := strings.SplitN(authHeader, " ", 2)
        if len(parts) != 2 || parts[0] != "Bearer" {
            c.JSON(401, gin.H{"error": "Invalid authorization format"})
            c.Abort()
            return
        }

        // 验证 Token
        claims, err := ParseToken(parts[1])
        if err != nil {
            c.JSON(401, gin.H{"error": "Invalid token"})
            c.Abort()
            return
        }

        // 将用户信息存入上下文
        c.Set("userID", claims.UserID)
        c.Set("username", claims.Username)
        c.Next()
    }
}
```

### 3.5 JWT 安全建议

```
1. 使用 HTTPS
   - 防止 Token 被窃取

2. 设置合理的过期时间
   - Access Token：15分钟 ~ 2小时
   - Refresh Token：7天 ~ 30天

3. 不要在 Payload 中存敏感信息
   - Payload 只是 Base64 编码，不是加密

4. 使用强密钥
   - 至少 256 位随机字符串

5. 实现 Token 刷新机制
   - Access Token 过期后用 Refresh Token 获取新的

6. 实现 Token 黑名单（可选）
   - 用于强制登出
```

## 4. OAuth 2.0 ⭐⭐

### 4.1 OAuth 2.0 概念

```
OAuth 2.0 = 授权框架

场景：第三方登录
- 用微信登录某 App
- 用 GitHub 登录某网站

角色：
1. Resource Owner（资源所有者）：用户
2. Client（客户端）：第三方应用
3. Authorization Server（授权服务器）：微信/GitHub
4. Resource Server（资源服务器）：存储用户数据的服务器
```

### 4.2 授权码模式（最常用）

```
流程：

1. 用户点击"用 GitHub 登录"
   → 跳转到 GitHub 授权页面

2. 用户同意授权
   → GitHub 重定向回应用，带上授权码
   https://myapp.com/callback?code=abc123

3. 应用用授权码换取 Access Token
   POST https://github.com/oauth/token
   {
       client_id: "xxx",
       client_secret: "xxx",
       code: "abc123"
   }
   → 返回 {access_token: "xxx"}

4. 应用用 Access Token 获取用户信息
   GET https://api.github.com/user
   Authorization: Bearer xxx
   → 返回用户信息
```

### 4.3 Go 实现 OAuth 2.0

```go
package main

import (
    "golang.org/x/oauth2"
    "golang.org/x/oauth2/github"
)

var githubOAuth = &oauth2.Config{
    ClientID:     "your-client-id",
    ClientSecret: "your-client-secret",
    RedirectURL:  "http://localhost:8080/callback",
    Scopes:       []string{"user:email"},
    Endpoint:     github.Endpoint,
}

// 1. 重定向到 GitHub 授权页面
func LoginHandler(c *gin.Context) {
    state := generateRandomState()
    session.Set("oauth_state", state)

    url := githubOAuth.AuthCodeURL(state)
    c.Redirect(302, url)
}

// 2. 处理回调
func CallbackHandler(c *gin.Context) {
    // 验证 state
    state := c.Query("state")
    if state != session.Get("oauth_state") {
        c.JSON(400, gin.H{"error": "Invalid state"})
        return
    }

    // 用授权码换取 Token
    code := c.Query("code")
    token, err := githubOAuth.Exchange(c, code)
    if err != nil {
        c.JSON(500, gin.H{"error": "Failed to exchange token"})
        return
    }

    // 用 Token 获取用户信息
    client := githubOAuth.Client(c, token)
    resp, _ := client.Get("https://api.github.com/user")

    // 解析用户信息，创建或登录用户
    // ...
}
```

## 5. 安全最佳实践 ⭐⭐⭐

### 5.1 密码安全

```go
import "golang.org/x/crypto/bcrypt"

// 加密密码
func HashPassword(password string) (string, error) {
    bytes, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
    return string(bytes), err
}

// 验证密码
func CheckPassword(password, hash string) bool {
    err := bcrypt.CompareHashAndPassword([]byte(hash), []byte(password))
    return err == nil
}

// 使用
hash, _ := HashPassword("user_password")
// 存储 hash 到数据库

// 验证
if CheckPassword(inputPassword, storedHash) {
    // 密码正确
}
```

### 5.2 安全响应头

```go
func SecurityHeaders() gin.HandlerFunc {
    return func(c *gin.Context) {
        // 防止 XSS
        c.Header("X-XSS-Protection", "1; mode=block")

        // 防止 MIME 类型嗅探
        c.Header("X-Content-Type-Options", "nosniff")

        // 防止点击劫持
        c.Header("X-Frame-Options", "DENY")

        // 强制 HTTPS
        c.Header("Strict-Transport-Security", "max-age=31536000; includeSubDomains")

        // CSP
        c.Header("Content-Security-Policy", "default-src 'self'")

        c.Next()
    }
}
```

### 5.3 限流防刷

```go
import "golang.org/x/time/rate"

// 创建限流器
var limiter = rate.NewLimiter(rate.Limit(10), 100)  // 每秒 10 个，桶大小 100

func RateLimitMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        if !limiter.Allow() {
            c.JSON(429, gin.H{"error": "Too many requests"})
            c.Abort()
            return
        }
        c.Next()
    }
}

// 按 IP 限流
// 注意：生产环境需要定期清理过期的限流器，避免内存泄漏
// 建议使用带过期时间的缓存（如 github.com/patrickmn/go-cache）
// 或限制 map 大小使用 LRU 策略
var limiters = make(map[string]*rate.Limiter)
var mu sync.Mutex

func GetLimiter(ip string) *rate.Limiter {
    mu.Lock()
    defer mu.Unlock()

    if limiter, exists := limiters[ip]; exists {
        return limiter
    }

    limiter := rate.NewLimiter(rate.Limit(1), 5)  // 每秒 1 个，桶大小 5
    limiters[ip] = limiter
    return limiter
}
```

## 6. 本章总结

### 必须记住的内容

```
常见攻击：
- SQL 注入：参数化查询防御
- XSS：输出转义防御
- CSRF：Token + SameSite Cookie 防御

JWT：
- Header.Payload.Signature
- 用于无状态认证
- 注意设置过期时间

安全实践：
- 使用 HTTPS
- 密码用 bcrypt 加密
- 设置安全响应头
- 实现限流
```

### 面试常见问题

```
Q: 什么是 SQL 注入？如何防御？
A: 用户输入被当作 SQL 执行。使用参数化查询防御。

Q: XSS 和 CSRF 的区别？
A: XSS 是注入恶意脚本，CSRF 是利用用户身份发送请求。

Q: JWT 的优缺点？
A: 优点：无状态、跨域。缺点：无法主动失效、Payload 不加密。

Q: 如何安全存储密码？
A: 使用 bcrypt 等慢哈希算法，不要用 MD5/SHA1。
```
