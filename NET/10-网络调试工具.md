## 1. curl 命令详解 ⭐⭐⭐

### 1.1 基本用法

```bash
# 最简单的 GET 请求
curl https://api.example.com/users

# 显示响应头
curl -i https://api.example.com/users

# 只显示响应头
curl -I https://api.example.com/users

# 显示详细信息（包括请求头）
curl -v https://api.example.com/users

# 静默模式（不显示进度）
curl -s https://api.example.com/users

# 保存响应到文件
curl -o response.json https://api.example.com/users

# 跟随重定向
curl -L https://example.com
```

### 1.2 HTTP 方法

```bash
# GET（默认）
curl https://api.example.com/users

# POST
curl -X POST https://api.example.com/users

# PUT
curl -X PUT https://api.example.com/users/1

# PATCH
curl -X PATCH https://api.example.com/users/1

# DELETE
curl -X DELETE https://api.example.com/users/1
```

### 1.3 发送数据

```bash
# POST JSON 数据
curl -X POST https://api.example.com/users \
    -H "Content-Type: application/json" \
    -d '{"name":"张三","email":"zhangsan@example.com"}'

# POST 表单数据
curl -X POST https://api.example.com/login \
    -d "username=admin&password=123456"

# 从文件读取数据
curl -X POST https://api.example.com/users \
    -H "Content-Type: application/json" \
    -d @user.json

# 上传文件
curl -X POST https://api.example.com/upload \
    -F "file=@photo.jpg"
```

### 1.4 设置请求头

```bash
# 设置单个头
curl -H "Authorization: Bearer token123" https://api.example.com/users

# 设置多个头
curl -H "Authorization: Bearer token123" \
     -H "Content-Type: application/json" \
     -H "Accept: application/json" \
     https://api.example.com/users

# 设置 User-Agent
curl -A "MyApp/1.0" https://api.example.com/users

# 设置 Cookie
curl -b "session=abc123" https://api.example.com/users

# 保存 Cookie 到文件
curl -c cookies.txt https://api.example.com/login

# 使用 Cookie 文件
curl -b cookies.txt https://api.example.com/dashboard
```

### 1.5 调试技巧

```bash
# 查看请求和响应的详细信息
curl -v https://api.example.com/users

# 只看请求时间
curl -w "DNS: %{time_namelookup}s\n连接: %{time_connect}s\n总时间: %{time_total}s\n" \
     -o /dev/null -s https://api.example.com/users

# 测试 HTTPS 证书
curl -v --cacert ca.crt https://api.example.com

# 忽略证书验证（仅测试用）
curl -k https://self-signed.example.com

# 设置超时
curl --connect-timeout 5 --max-time 10 https://api.example.com
```

### 1.6 常用组合

```bash
# 完整的 API 测试
curl -X POST https://api.example.com/users \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer token123" \
    -d '{"name":"张三","age":25}' \
    -w "\n状态码: %{http_code}\n" \
    -s | jq .

# 测试接口响应时间
curl -o /dev/null -s -w "响应时间: %{time_total}s\n" https://api.example.com

# 下载文件并显示进度
curl -# -O https://example.com/file.zip
```

## 2. Postman 使用 ⭐⭐⭐

### 2.1 基本功能

```
Postman 是图形化的 API 测试工具

主要功能：
1. 发送各种 HTTP 请求
2. 管理请求集合
3. 设置环境变量
4. 自动化测试
5. 生成 API 文档
6. 团队协作
```

### 2.2 发送请求

```
1. 选择 HTTP 方法（GET/POST/PUT/DELETE）
2. 输入 URL
3. 设置请求头（Headers 标签）
4. 设置请求体（Body 标签）
   - form-data：表单数据
   - x-www-form-urlencoded：URL 编码表单
   - raw：原始数据（JSON/XML）
   - binary：二进制文件
5. 点击 Send 发送请求
```

### 2.3 环境变量

```
用途：在不同环境（开发/测试/生产）切换

设置变量：
{{base_url}} = http://localhost:8080

使用变量：
GET {{base_url}}/api/users

环境示例：
开发环境：base_url = http://localhost:8080
测试环境：base_url = http://test.example.com
生产环境：base_url = http://api.example.com
```

### 2.4 测试脚本

```javascript
// Tests 标签中编写测试脚本

// 检查状态码
pm.test("状态码是 200", function () {
    pm.response.to.have.status(200);
});

// 检查响应时间
pm.test("响应时间小于 500ms", function () {
    pm.expect(pm.response.responseTime).to.be.below(500);
});

// 检查响应体
pm.test("返回正确的用户名", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData.name).to.eql("张三");
});

// 保存响应数据到变量
var jsonData = pm.response.json();
pm.environment.set("user_id", jsonData.id);
```

### 2.5 集合和文件夹

```
组织 API 请求：

用户管理/
├── 获取用户列表    GET /users
├── 获取单个用户    GET /users/:id
├── 创建用户        POST /users
├── 更新用户        PUT /users/:id
└── 删除用户        DELETE /users/:id

文章管理/
├── 获取文章列表    GET /articles
├── 获取单篇文章    GET /articles/:id
└── ...
```

## 3. Wireshark 抓包 ⭐⭐⭐

### 3.1 Wireshark 简介

```
Wireshark = 网络协议分析器

功能：
- 捕获网络数据包
- 分析各层协议
- 过滤和搜索数据包
- 导出和保存抓包结果

适用场景：
- 分析网络问题
- 学习网络协议
- 安全分析
- 性能调优
```

### 3.2 基本使用

```
1. 选择网络接口
   - eth0：有线网卡
   - wlan0：无线网卡
   - lo：本地回环

2. 开始捕获
   - 点击鲨鱼鳍图标开始
   - 点击红色方块停止

3. 查看数据包
   - 上方：数据包列表
   - 中间：协议详情
   - 下方：原始数据（十六进制）
```

### 3.3 过滤器

```
捕获过滤器（抓包前设置）：
host 192.168.1.100        # 指定主机
port 80                   # 指定端口
tcp port 443              # TCP 443 端口
src host 192.168.1.100    # 源地址
dst port 80               # 目标端口

显示过滤器（抓包后过滤）：
ip.addr == 192.168.1.100  # IP 地址
tcp.port == 80            # TCP 端口
http                      # HTTP 协议
tcp.flags.syn == 1        # TCP SYN 包
http.request.method == "POST"  # HTTP POST 请求

常用过滤器：
http                      # 所有 HTTP 流量
tcp.port == 8080          # 8080 端口
ip.src == 192.168.1.100   # 源 IP
http.request              # HTTP 请求
http.response             # HTTP 响应
tcp.analysis.retransmission  # TCP 重传
```

### 3.4 分析 TCP 三次握手

```
过滤器：tcp.flags.syn == 1 or tcp.flags.ack == 1

三次握手数据包：
1. SYN        客户端 → 服务端
   Flags: SYN
   Seq: 0

2. SYN+ACK    服务端 → 客户端
   Flags: SYN, ACK
   Seq: 0, Ack: 1

3. ACK        客户端 → 服务端
   Flags: ACK
   Seq: 1, Ack: 1
```

### 3.5 分析 HTTP 请求

```
过滤器：http

HTTP 请求包：
- 请求行：GET /api/users HTTP/1.1
- 请求头：Host, User-Agent, Accept 等
- 请求体：POST 数据

HTTP 响应包：
- 状态行：HTTP/1.1 200 OK
- 响应头：Content-Type, Content-Length 等
- 响应体：JSON 数据

右键 → Follow → HTTP Stream
可以看到完整的 HTTP 会话
```

## 4. tcpdump 命令行抓包 ⭐⭐

### 4.1 基本用法

```bash
# 抓取所有数据包
sudo tcpdump

# 指定网卡
sudo tcpdump -i eth0

# 指定主机
sudo tcpdump host 192.168.1.100

# 指定端口
sudo tcpdump port 80

# 指定协议
sudo tcpdump tcp
sudo tcpdump udp
sudo tcpdump icmp
```

### 4.2 常用选项

```bash
# 显示详细信息
sudo tcpdump -v
sudo tcpdump -vv   # 更详细
sudo tcpdump -vvv  # 最详细

# 显示数据包内容（ASCII）
sudo tcpdump -A

# 显示数据包内容（十六进制）
sudo tcpdump -X

# 不解析主机名
sudo tcpdump -n

# 不解析端口名
sudo tcpdump -nn

# 限制抓包数量
sudo tcpdump -c 100

# 保存到文件
sudo tcpdump -w capture.pcap

# 读取文件
sudo tcpdump -r capture.pcap
```

### 4.3 过滤表达式

```bash
# 源地址
sudo tcpdump src 192.168.1.100

# 目标地址
sudo tcpdump dst 192.168.1.100

# 源端口
sudo tcpdump src port 80

# 目标端口
sudo tcpdump dst port 80

# 组合条件
sudo tcpdump 'host 192.168.1.100 and port 80'
sudo tcpdump 'src 192.168.1.100 or dst 192.168.1.100'
sudo tcpdump 'port 80 and not host 192.168.1.1'

# TCP 标志
sudo tcpdump 'tcp[tcpflags] & tcp-syn != 0'  # SYN 包
sudo tcpdump 'tcp[tcpflags] & tcp-rst != 0'  # RST 包
```

### 4.4 实用示例

```bash
# 抓取 HTTP 请求
sudo tcpdump -i eth0 -A -s 0 'tcp port 80 and (((ip[2:2] - ((ip[0]&0xf)<<2)) - ((tcp[12]&0xf0)>>2)) != 0)'

# 抓取 DNS 查询
sudo tcpdump -i eth0 port 53

# 抓取 HTTPS 握手
sudo tcpdump -i eth0 'tcp port 443 and (tcp[((tcp[12] & 0xf0) >> 2)] = 0x16)'

# 抓取特定主机的所有流量并保存
sudo tcpdump -i eth0 host 192.168.1.100 -w host.pcap

# 实时查看 HTTP 请求
sudo tcpdump -i eth0 -A -s 0 'tcp port 80' | grep -E 'GET|POST|HTTP'
```

## 5. 其他实用工具 ⭐⭐

### 5.1 netcat (nc)

```bash
# 端口扫描
nc -zv 192.168.1.100 80
nc -zv 192.168.1.100 1-1000

# 简单的 TCP 服务器
nc -l 8080

# TCP 客户端
nc 192.168.1.100 8080

# 发送 HTTP 请求
echo -e "GET / HTTP/1.1\r\nHost: example.com\r\n\r\n" | nc example.com 80

# 文件传输
# 接收端
nc -l 8080 > file.txt
# 发送端
nc 192.168.1.100 8080 < file.txt
```

### 5.2 telnet

```bash
# 测试端口连通性
telnet 192.168.1.100 80

# 手动发送 HTTP 请求
telnet example.com 80
GET / HTTP/1.1
Host: example.com
（按两次回车）
```

### 5.3 nmap

```bash
# 端口扫描
nmap 192.168.1.100

# 扫描指定端口
nmap -p 80,443,8080 192.168.1.100

# 扫描端口范围
nmap -p 1-1000 192.168.1.100

# 扫描整个网段
nmap 192.168.1.0/24

# 服务版本检测
nmap -sV 192.168.1.100

# 操作系统检测
nmap -O 192.168.1.100
```

### 5.4 dig / nslookup

```bash
# DNS 查询
dig example.com
nslookup example.com

# 查询特定记录类型
dig example.com A      # A 记录
dig example.com AAAA   # AAAA 记录
dig example.com MX     # MX 记录
dig example.com NS     # NS 记录
dig example.com TXT    # TXT 记录

# 指定 DNS 服务器
dig @8.8.8.8 example.com

# 反向解析
dig -x 8.8.8.8
```

## 6. 调试场景实战 ⭐⭐⭐

### 6.1 接口返回 500 错误

```bash
# 1. 用 curl 查看详细响应
curl -v https://api.example.com/users

# 2. 检查请求头是否正确
curl -H "Content-Type: application/json" \
     -H "Authorization: Bearer token" \
     https://api.example.com/users

# 3. 查看服务器日志
tail -f /var/log/app/error.log
```

### 6.2 连接超时

```bash
# 1. 测试网络连通性
ping 192.168.1.100

# 2. 测试端口是否开放
nc -zv 192.168.1.100 80
telnet 192.168.1.100 80

# 3. 追踪路由
traceroute 192.168.1.100

# 4. 检查防火墙
sudo iptables -L -n
```

### 6.3 HTTPS 证书问题

```bash
# 1. 查看证书信息
openssl s_client -connect example.com:443

# 2. 检查证书有效期
echo | openssl s_client -connect example.com:443 2>/dev/null | openssl x509 -noout -dates

# 3. 验证证书链
curl -v https://example.com 2>&1 | grep -A 5 "SSL certificate"
```

### 6.4 DNS 解析问题

```bash
# 1. 检查 DNS 解析
dig example.com
nslookup example.com

# 2. 检查本地 DNS 配置
cat /etc/resolv.conf

# 3. 使用其他 DNS 服务器测试
dig @8.8.8.8 example.com

# 4. 清除 DNS 缓存
# Mac
sudo dscacheutil -flushcache
# Linux
sudo systemd-resolve --flush-caches
```

## 7. 本章总结

### 必须记住的命令

```bash
# curl 常用
curl -v URL                    # 详细信息
curl -X POST -d 'data' URL     # POST 请求
curl -H "Header: value" URL    # 设置请求头

# tcpdump 常用
sudo tcpdump -i eth0 port 80   # 抓取 80 端口
sudo tcpdump -w file.pcap      # 保存到文件
sudo tcpdump host 192.168.1.1  # 指定主机

# 网络诊断
ping IP                        # 测试连通性
telnet IP PORT                 # 测试端口
traceroute IP                  # 追踪路由
dig domain                     # DNS 查询
```

### 工具选择

```
场景 → 工具：
- 快速测试 API → curl
- 复杂 API 测试 → Postman
- 深入分析协议 → Wireshark
- 服务器抓包 → tcpdump
- 端口扫描 → nmap
- DNS 问题 → dig
```
