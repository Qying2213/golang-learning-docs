## 1. 什么是计算机网络？⭐⭐⭐

### 1.1 最简单的理解

```
计算机网络 = 让不同的计算机能够互相通信

就像：
- 电话网络：让不同的电话能打电话
- 快递网络：让不同的地方能寄快递
- 计算机网络：让不同的电脑能传数据
```

### 1.2 日常例子

```
你在浏览器输入 www.baidu.com

1. 你的电脑（客户端）
   ↓
2. 通过网络
   ↓
3. 到达百度的服务器
   ↓
4. 服务器返回网页
   ↓
5. 你看到了百度首页

这整个过程就是计算机网络在工作！
```

### 1.3 网络的组成

```
┌─────────────────────────────────────────────────┐
│                  互联网                          │
│  ┌──────────┐      ┌──────────┐                │
│  │ 你的电脑  │ ←→  │ 路由器    │                │
│  └──────────┘      └──────────┘                │
│                         ↓                       │
│                    ┌──────────┐                │
│                    │ 交换机    │                │
│                    └──────────┘                │
│                         ↓                       │
│                    ┌──────────┐                │
│                    │ 服务器    │                │
│                    └──────────┘                │
└─────────────────────────────────────────────────┘

关键设备：
- 电脑/手机：终端设备
- 路由器：连接不同网络
- 交换机：连接同一网络内的设备
- 服务器：提供服务的电脑
```

## 2. 网络分层模型 ⭐⭐⭐

### 2.1 为什么要分层？

```
想象寄快递：

不分层（混乱）：
你 → 直接把包裹送到对方家 ❌
- 太复杂了！
- 每个人都要知道所有细节

分层（清晰）：
你 → 打包 → 贴地址 → 快递员 → 分拣中心 → 派送 → 对方 ✅
- 每个环节只做自己的事
- 简单、高效、可维护
```

### 2.2 OSI 七层模型（理论模型）

```
┌─────────────────────────────────────────────────┐
│  7. 应用层 (Application)                         │
│     HTTP、FTP、SMTP、DNS                        │
│     你的程序直接使用的协议                        │
├─────────────────────────────────────────────────┤
│  6. 表示层 (Presentation)                       │
│     数据格式转换、加密解密                        │
├─────────────────────────────────────────────────┤
│  5. 会话层 (Session)                            │
│     建立、管理、终止会话                          │
├─────────────────────────────────────────────────┤
│  4. 传输层 (Transport)                          │
│     TCP、UDP                                    │
│     端到端的数据传输                             │
├─────────────────────────────────────────────────┤
│  3. 网络层 (Network)                            │
│     IP、路由                                    │
│     寻址和路由选择                               │
├─────────────────────────────────────────────────┤
│  2. 数据链路层 (Data Link)                      │
│     以太网、WiFi                                │
│     相邻节点间的数据传输                          │
├─────────────────────────────────────────────────┤
│  1. 物理层 (Physical)                           │
│     网线、光纤、无线电波                          │
│     物理信号传输                                 │
└─────────────────────────────────────────────────┘
```

### 2.3 TCP/IP 四层模型（实际使用）

```
┌─────────────────────────────────────────────────┐
│  应用层 (Application Layer)                      │
│  HTTP、HTTPS、FTP、SMTP、DNS、SSH               │
│  你写的 Go 程序在这一层                          │
├─────────────────────────────────────────────────┤
│  传输层 (Transport Layer)                       │
│  TCP、UDP                                       │
│  负责端到端的通信                                │
├─────────────────────────────────────────────────┤
│  网络层 (Internet Layer)                        │
│  IP、ICMP                                       │
│  负责寻址和路由                                  │
├─────────────────────────────────────────────────┤
│  网络接口层 (Network Interface Layer)            │
│  以太网、WiFi、ARP、网卡驱动                     │
│  负责物理传输（ARP 用于 IP 到 MAC 地址解析）      │
└─────────────────────────────────────────────────┘

记忆口诀：应传网接（应该传网接）
```

### 2.4 数据传输过程

```
发送数据（封装）：

应用层：HTTP 请求
   ↓ 加上 HTTP 头
传输层：TCP 段
   ↓ 加上 TCP 头（端口号）
网络层：IP 包
   ↓ 加上 IP 头（IP 地址）
网络接口层：以太网帧
   ↓ 加上 MAC 地址
物理层：01010101...（电信号）

就像寄快递：
你的东西 → 装箱 → 贴快递单 → 装车 → 运输
```

### 📝 练习题 2.1

**问题**：当你在浏览器访问 `http://www.baidu.com` 时，数据经过了哪些层？

<details>
<summary>点击查看答案</summary>

**答案**：

```
你的电脑（发送）：
1. 应用层：浏览器生成 HTTP 请求
2. 传输层：TCP 添加端口号（目标端口 80）
3. 网络层：IP 添加百度服务器的 IP 地址
4. 网络接口层：添加 MAC 地址，发送到路由器

中间路由器：
- 网络层：根据 IP 地址转发数据包

百度服务器（接收）：
1. 网络接口层：接收数据帧
2. 网络层：检查 IP 地址（是发给我的）
3. 传输层：检查端口号（80 端口）
4. 应用层：Web 服务器处理 HTTP 请求

然后反向返回响应！
```

</details>

## 3. IP 地址 ⭐⭐⭐

### 3.1 什么是 IP 地址？

```
IP 地址 = 计算机在网络中的地址

就像：
- 你的家庭地址：北京市朝阳区xxx街道xxx号
- 计算机的地址：192.168.1.100

有了地址，数据才知道往哪里发！
```

### 3.2 IPv4 地址格式

```
192.168.1.100
│   │   │ │
│   │   │ └─ 第4段（0-255）
│   │   └─── 第3段（0-255）
│   └─────── 第2段（0-255）
└─────────── 第1段（0-255）

每段是 8 位二进制数（0-255）
总共 32 位

例子：
192.168.1.100
= 11000000.10101000.00000001.01100100（二进制）
```

### 3.3 特殊的 IP 地址

```
127.0.0.1
- 本地回环地址
- 指向自己的电脑
- localhost 就是这个地址

0.0.0.0
- 表示"所有地址"
- 服务器监听 0.0.0.0 = 监听所有网卡

192.168.x.x
- 私有地址（内网）
- 不能在互联网上直接访问
- 你家路由器分配的就是这种地址

10.x.x.x
- 也是私有地址
- 公司内网常用

255.255.255.255
- 广播地址
- 发给所有人
```

### 3.4 公网 IP vs 内网 IP

```
公网 IP（Public IP）：
- 全球唯一
- 可以在互联网上访问
- 例如：百度的服务器 IP

内网 IP（Private IP）：
- 局域网内使用
- 不能直接在互联网访问
- 例如：你家电脑的 192.168.1.100

┌─────────────────────────────────────────┐
│           互联网（公网）                  │
│                                          │
│  百度服务器：220.181.38.148（公网IP）    │
│                                          │
│  你家路由器：123.45.67.89（公网IP）      │
│       │                                  │
└───────┼──────────────────────────────────┘
        │
        ↓
┌─────────────────────────────────────────┐
│      你家局域网（内网）                   │
│                                          │
│  你的电脑：192.168.1.100（内网IP）       │
│  你的手机：192.168.1.101（内网IP）       │
│  智能电视：192.168.1.102（内网IP）       │
└─────────────────────────────────────────┘
```

### 📝 练习题 3.1

**问题**：为什么你的电脑 IP 是 192.168.1.100，但能访问百度？

<details>
<summary>点击查看答案</summary>

**答案**：

通过 **NAT（网络地址转换）**！

```
1. 你的电脑（192.168.1.100）发送请求
2. 路由器收到请求
3. 路由器把源 IP 改成公网 IP（123.45.67.89）
4. 发送到百度服务器
5. 百度返回数据到 123.45.67.89
6. 路由器收到数据
7. 路由器把目标 IP 改成 192.168.1.100
8. 转发给你的电脑

这样，内网 IP 就能访问互联网了！
```

</details>

## 4. 端口号 ⭐⭐⭐

### 4.1 什么是端口？

```
IP 地址 = 找到哪台电脑
端口号 = 找到电脑上的哪个程序

比喻：
- IP 地址 = 小区地址
- 端口号 = 门牌号

192.168.1.100:8080
│              │
│              └─ 端口号（8080）
└─ IP 地址
```

### 4.2 端口号范围

```
0 - 65535（总共 65536 个端口）

分类：
0-1023：系统端口（需要管理员权限）
  - 80：HTTP
  - 443：HTTPS
  - 22：SSH
  - 3306：MySQL
  - 6379：Redis

1024-49151：注册端口
  - 8080：常用的 Web 服务
  - 8000：开发服务器
  - 3000：Node.js 开发

49152-65535：动态端口
  - 系统自动分配
  - 客户端连接时使用
```

### 4.3 常用端口

```
Web 服务：
80   - HTTP（网页）
443  - HTTPS（加密网页）
8080 - HTTP 备用端口

数据库：
3306 - MySQL
5432 - PostgreSQL
6379 - Redis
27017 - MongoDB

其他：
22   - SSH（远程登录）
21   - FTP（文件传输）
25   - SMTP（发邮件）
53   - DNS（域名解析）
```

### 4.4 Go 程序中的端口

```go
package main

import (
    "fmt"
    "net/http"
)

func main() {
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintf(w, "Hello World")
    })

    // 监听 8080 端口
    http.ListenAndServe(":8080", nil)
    //                   └─ 端口号
}

// 访问：http://localhost:8080
```

### 📝 练习题 4.1

**问题**：为什么开发时常用 8080 端口，而不是 80 端口？

<details>
<summary>点击查看答案</summary>

**答案**：

1. **权限问题**：
   - 80 端口是系统端口（0-1023）
   - 需要管理员权限才能监听
   - 8080 是普通端口，不需要特殊权限

2. **避免冲突**：
   - 80 端口可能已经被其他程序占用
   - 8080 通常是空闲的

3. **开发习惯**：
   - 8080 是约定俗成的开发端口
   - 容易记忆

```bash
# 监听 80 端口（需要 sudo）
sudo ./myapp -port 80

# 监听 8080 端口（不需要 sudo）
./myapp -port 8080
```

</details>

## 5. 域名和 DNS ⭐⭐⭐

### 5.1 为什么需要域名？

```
问题：IP 地址难记

220.181.38.148  ← 这是百度的 IP，你记得住吗？❌
www.baidu.com   ← 这个好记！✅

域名 = IP 地址的别名
就像：手机通讯录里存的是"张三"，而不是手机号
```

### 5.2 域名的结构

```
www.baidu.com
│   │     │
│   │     └─ 顶级域名（TLD）：com、cn、org、net
│   └─────── 二级域名：baidu
└─────────── 三级域名（子域名）：www

完整的域名结构：
www.baidu.com.
│   │     │   │
│   │     │   └─ 根域名（通常省略）
│   │     └───── 顶级域名
│   └─────────── 二级域名
└─────────────── 子域名

例子：
api.example.com  ← API 服务器
www.example.com  ← 网站
mail.example.com ← 邮件服务器
```

### 5.3 DNS 解析过程

```
你输入：www.baidu.com

1. 浏览器查询本地缓存
   ├─ 有缓存 → 直接使用 ✓
   └─ 没缓存 → 继续

2. 查询操作系统缓存
   ├─ 有缓存 → 返回 ✓
   └─ 没缓存 → 继续

3. 查询本地 DNS 服务器（通常是路由器）
   ├─ 有缓存 → 返回 ✓
   └─ 没缓存 → 继续

4. 查询根 DNS 服务器
   └─ 返回：.com 的 DNS 服务器地址

5. 查询 .com DNS 服务器
   └─ 返回：baidu.com 的 DNS 服务器地址

6. 查询 baidu.com DNS 服务器
   └─ 返回：www.baidu.com 的 IP 地址

7. 得到 IP：220.181.38.148

8. 浏览器连接这个 IP
```

### 5.4 查看 DNS 解析

```bash
# 查询域名的 IP 地址
nslookup www.baidu.com

# 输出：
Server:  192.168.1.1
Address: 192.168.1.1#53

Non-authoritative answer:
www.baidu.com canonical name = www.a.shifen.com.
Name: www.a.shifen.com
Address: 220.181.38.148

# 或者用 dig 命令（更详细）
dig www.baidu.com

# 或者用 host 命令
host www.baidu.com
```

### 5.5 /etc/hosts 文件

```bash
# 本地 DNS 配置文件
cat /etc/hosts

# 内容示例：
127.0.0.1   localhost
127.0.0.1   myapp.local
192.168.1.100   dev.example.com

# 作用：
# 在查询 DNS 之前，先查这个文件
# 可以用来：
# 1. 本地开发测试
# 2. 屏蔽广告网站
# 3. 加速常用网站访问
```

### 📝 练习题 5.1

**问题**：为什么有时候网站打不开，但 ping IP 地址可以？

<details>
<summary>点击查看答案</summary>

**答案**：

可能是 **DNS 解析失败**！

```
情况1：DNS 服务器故障
- 域名无法解析成 IP
- 但直接用 IP 可以访问

情况2：DNS 缓存问题
- 缓存了错误的 IP
- 清除缓存可以解决

情况3：DNS 劫持
- DNS 被篡改
- 解析到错误的 IP

解决方法：
# 1. 清除 DNS 缓存
sudo dscacheutil -flushcache  # Mac
ipconfig /flushdns             # Windows

# 2. 更换 DNS 服务器
# 改成公共 DNS：
# 8.8.8.8（Google）
# 114.114.114.114（国内）

# 3. 直接用 IP 访问
curl http://220.181.38.148
```

</details>
