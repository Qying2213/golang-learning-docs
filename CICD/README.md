## 💡 为什么要学 CI/CD？

**没有 CI/CD 的开发流程：**
```
写代码 → 手动测试 → 手动打包 → 手动上传服务器 → 手动重启服务
   ↓
  累！容易出错！效率低！
```

**有 CI/CD 的开发流程：**
```
写代码 → git push → 自动测试 → 自动打包 → 自动部署
   ↓
  爽！稳定！高效！
```

## 🎯 学完能做什么？

1. 代码 push 后自动运行测试
2. 自动构建 Docker 镜像
3. 自动推送到镜像仓库
4. 自动部署到服务器
5. 出问题自动通知
