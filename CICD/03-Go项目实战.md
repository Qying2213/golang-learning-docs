## 1. 项目结构

```
myapp/
├── main.go
├── go.mod
├── go.sum
├── config/

├── handler/
│   └── user.go
├── model/
│   └── user.go
├── middleware/
│   └── auth.go
├── Dockerfile
└── .github/
    └── workflows/
```

## 2. 完整的 CI/CD 配置

### 2.1 基础版（适合学习）

```yaml
# .github/workflows/ci.yml
name: Go CI/CD

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  # ==================== 测试 ====================
  test:
    name: 测试
    runs-on: ubuntu-latest
    
    steps:
      - name: 拉取代码
        uses: actions/checkout@v3
      
      - name: 设置 Go 环境
        uses: actions/setup-go@v4
        with:
          go-version: '1.21'
      
      - name: 下载依赖
        run: go mod download
      
      - name: 运行测试
        run: go test -v ./...
      
      - name: 编译检查
        run: go build -v ./...

  # ==================== 构建 ====================
  build:
    name: 构建
    runs-on: ubuntu-latest
    needs: test  # 测试通过后才构建
    
    steps:
      - name: 拉取代码
        uses: actions/checkout@v3
      
      - name: 设置 Go 环境
        uses: actions/setup-go@v4
        with:
          go-version: '1.21'
      
      - name: 编译为 Linux 可执行文件
        run: |
          CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o app main.go
      
      - name: 上传构建产物
        uses: actions/upload-artifact@v3
        with:
          name: app
          path: app
```

### 2.2 进阶版（生产可用）

```yaml
# .github/workflows/ci.yml
name: Go CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
    tags: [ 'v*' ]  # 支持版本标签
  pull_request:
    branches: [ main ]

env:
  GO_VERSION: '1.21'
  APP_NAME: myapp

jobs:
  # ==================== 代码检查 ====================
  lint:
    name: 代码检查
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - uses: actions/setup-go@v4
        with:
          go-version: ${{ env.GO_VERSION }}
      
      - name: golangci-lint
        uses: golangci/golangci-lint-action@v3
        with:
          version: latest
          args: --timeout=5m

  # ==================== 单元测试 ====================
  test:
    name: 单元测试
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - uses: actions/setup-go@v4
        with:
          go-version: ${{ env.GO_VERSION }}
      
      - name: 缓存 Go 模块
        uses: actions/cache@v3
        with:
          path: |
            ~/go/pkg/mod
            ~/.cache/go-build
          key: ${{ runner.os }}-go-${{ hashFiles('**/go.sum') }}
          restore-keys: |
            ${{ runner.os }}-go-
      
      - name: 下载依赖
        run: go mod download
      
      - name: 运行测试（带覆盖率）
        run: |
          go test -v -race -coverprofile=coverage.out -covermode=atomic ./...
      
      - name: 上传覆盖率到 Codecov
        uses: codecov/codecov-action@v3
        with:
          files: ./coverage.out
          fail_ci_if_error: false

  # ==================== 构建 ====================
  build:
    name: 构建
    runs-on: ubuntu-latest
    needs: [lint, test]
    
    strategy:
      matrix:
        goos: [linux, darwin, windows]
        goarch: [amd64, arm64]
        exclude:
          - goos: windows
            goarch: arm64
    
    steps:
      - uses: actions/checkout@v3
      
      - uses: actions/setup-go@v4
        with:
          go-version: ${{ env.GO_VERSION }}
      
      - name: 构建
        env:
          GOOS: ${{ matrix.goos }}
          GOARCH: ${{ matrix.goarch }}
        run: |
          OUTPUT_NAME=${{ env.APP_NAME }}-${{ matrix.goos }}-${{ matrix.goarch }}
          if [ "${{ matrix.goos }}" = "windows" ]; then
            OUTPUT_NAME="${OUTPUT_NAME}.exe"
          fi
          CGO_ENABLED=0 go build -ldflags="-s -w" -o ${OUTPUT_NAME} main.go
      
      - name: 上传构建产物
        uses: actions/upload-artifact@v3
        with:
          name: ${{ env.APP_NAME }}-${{ matrix.goos }}-${{ matrix.goarch }}
          path: ${{ env.APP_NAME }}-*

  # ==================== Docker 镜像 ====================
  docker:
    name: Docker 镜像
    runs-on: ubuntu-latest
    needs: [lint, test]
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    
    steps:
      - uses: actions/checkout@v3
      
      - name: 设置 Docker Buildx
        uses: docker/setup-buildx-action@v2
      
      - name: 登录 Docker Hub
        uses: docker/login-action@v2
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}
      
      - name: 提取元数据
        id: meta
        uses: docker/metadata-action@v4
        with:
          images: ${{ secrets.DOCKER_USERNAME }}/${{ env.APP_NAME }}
          tags: |
            type=ref,event=branch
            type=sha,prefix=
            type=raw,value=latest
      
      - name: 构建并推送
        uses: docker/build-push-action@v4
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # ==================== 部署 ====================
  deploy:
    name: 部署到服务器
    runs-on: ubuntu-latest
    needs: docker
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    
    steps:
      - name: 部署到服务器
        uses: appleboy/ssh-action@v1.0.0
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SERVER_SSH_KEY }}
          script: |
            cd /opt/myapp
            docker pull ${{ secrets.DOCKER_USERNAME }}/${{ env.APP_NAME }}:latest
            docker stop ${{ env.APP_NAME }} || true
            docker rm ${{ env.APP_NAME }} || true
            docker run -d \
              --name ${{ env.APP_NAME }} \
              -p 8080:8080 \
              --restart unless-stopped \
              ${{ secrets.DOCKER_USERNAME }}/${{ env.APP_NAME }}:latest
```

## 3. 配置详解

### 3.1 代码检查（Lint）

```yaml
- name: golangci-lint
  uses: golangci/golangci-lint-action@v3
  with:
    version: latest
    args: --timeout=5m
```

**golangci-lint 是什么？**
- Go 语言最流行的代码检查工具
- 集成了几十种检查器
- 能发现潜在 bug、代码风格问题

**本地也可以用：**
```bash
# 安装
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest

# 运行
golangci-lint run
```

### 3.2 测试覆盖率

```yaml
- name: 运行测试（带覆盖率）
  run: |
    go test -v -race -coverprofile=coverage.out -covermode=atomic ./...
```

**参数解释：**
- `-v`：显示详细输出
- `-race`：检测数据竞争
- `-coverprofile=coverage.out`：生成覆盖率报告
- `-covermode=atomic`：原子模式，适合并发测试
- `./...`：测试所有包

### 3.3 多平台构建（Matrix）

```yaml
strategy:
  matrix:
    goos: [linux, darwin, windows]
    goarch: [amd64, arm64]
    exclude:
      - goos: windows
        goarch: arm64
```

**效果：** 自动构建 5 个版本
- linux-amd64（Linux 64位）
- linux-arm64（Linux ARM）
- darwin-amd64（Mac Intel）
- darwin-arm64（Mac M1/M2）
- windows-amd64（Windows 64位）

### 3.4 条件执行

```yaml
if: github.event_name == 'push' && github.ref == 'refs/heads/main'
```

**含义：** 只有 push 到 main 分支时才执行

**常用条件：**
```yaml
# 只在 main 分支
if: github.ref == 'refs/heads/main'

# 只在 tag
if: startsWith(github.ref, 'refs/tags/')

# 只在 PR
if: github.event_name == 'pull_request'

# 前一个 job 成功
if: success()

# 前一个 job 失败
if: failure()

# 总是执行
if: always()
```

## 4. 需要配置的 Secrets

在 GitHub 仓库设置中添加以下 Secrets：

| Secret 名称 | 说明 | 示例 |
|-------------|------|------|
| `DOCKER_USERNAME` | Docker Hub 用户名 | `yourname` |
| `DOCKER_PASSWORD` | Docker Hub 密码或 Token | `dckr_pat_xxx` |
| `SERVER_HOST` | 服务器 IP | `192.168.1.100` |
| `SERVER_USER` | SSH 用户名 | `root` |
| `SERVER_SSH_KEY` | SSH 私钥 | `-----BEGIN...` |

**设置步骤：**
1. 进入 GitHub 仓库
2. Settings → Secrets and variables → Actions
3. New repository secret
4. 填写 Name 和 Value

## 5. Dockerfile 配置

### 5.1 基础版

```dockerfile
FROM golang:1.21-alpine AS builder

WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download

COPY . .
RUN CGO_ENABLED=0 go build -o main .

FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/main .

EXPOSE 8080
CMD ["./main"]
```

### 5.2 优化版（更小的镜像）

```dockerfile
# 构建阶段
FROM golang:1.21-alpine AS builder

WORKDIR /app

# 先复制依赖文件，利用缓存
COPY go.mod go.sum ./
RUN go mod download

# 复制源码并编译
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o main .

# 运行阶段
FROM scratch

# 复制时区信息（如果需要）
COPY --from=builder /usr/share/zoneinfo /usr/share/zoneinfo
# 复制 CA 证书（如果需要 HTTPS）
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/

WORKDIR /app
COPY --from=builder /app/main .

EXPOSE 8080
ENTRYPOINT ["./main"]
```

**镜像大小对比：**
```
基础版（alpine）：~30MB
优化版（scratch）：~10MB
```

## 6. 完整示例项目

### 6.1 main.go

```go
package main

import (
    "fmt"
    "log"
    "net/http"
    "os"
)

func main() {
    port := os.Getenv("PORT")
    if port == "" {
        port = "8080"
    }

    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintf(w, "Hello, CI/CD! Version: %s", getVersion())
    })

    http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
        fmt.Fprint(w, "OK")
    })

    log.Printf("Server starting on port %s", port)
    log.Fatal(http.ListenAndServe(":"+port, nil))
}

func getVersion() string {
    version := os.Getenv("APP_VERSION")
    if version == "" {
        return "dev"
    }
    return version
}
```

### 6.2 main_test.go

```go
package main

import (
    "net/http"
    "net/http/httptest"
    "testing"
)

func TestHealthEndpoint(t *testing.T) {
    req, err := http.NewRequest("GET", "/health", nil)
    if err != nil {
        t.Fatal(err)
    }

    rr := httptest.NewRecorder()
    handler := http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
        w.Write([]byte("OK"))
    })

    handler.ServeHTTP(rr, req)

    if status := rr.Code; status != http.StatusOK {
        t.Errorf("handler returned wrong status code: got %v want %v",
            status, http.StatusOK)
    }

    expected := "OK"
    if rr.Body.String() != expected {
        t.Errorf("handler returned unexpected body: got %v want %v",
            rr.Body.String(), expected)
    }
}

func TestGetVersion(t *testing.T) {
    // 测试默认值
    version := getVersion()
    if version != "dev" {
        t.Errorf("expected 'dev', got '%s'", version)
    }
}
```

## 8. 常见问题

### Q1：测试失败怎么办？

```yaml
# 查看详细日志
- name: 运行测试
  run: go test -v ./...
```

在 GitHub Actions 页面查看具体哪个测试失败了。

### Q2：构建太慢怎么办？

添加缓存：
```yaml
- name: 缓存 Go 模块
  uses: actions/cache@v3
  with:
    path: |
      ~/go/pkg/mod
      ~/.cache/go-build
    key: ${{ runner.os }}-go-${{ hashFiles('**/go.sum') }}
```

### Q3：Docker 推送失败？

检查 Secrets 是否正确配置：
- `DOCKER_USERNAME`
- `DOCKER_PASSWORD`（建议用 Access Token 而不是密码）

### Q4：SSH 部署失败？

1. 检查 `SERVER_SSH_KEY` 是否是私钥（不是公钥）
2. 确保服务器的 `~/.ssh/authorized_keys` 包含对应的公钥
3. 检查服务器防火墙是否开放 22 端口

## 10. 本章小结

```
Go 项目 CI/CD 流水线：
├── lint：代码检查（golangci-lint）
├── test：单元测试 + 覆盖率
├── build：多平台编译
├── docker：构建并推送镜像
└── deploy：SSH 部署到服务器

关键配置：
├── 缓存：加速构建
├── Matrix：多平台构建
├── Secrets：存储敏感信息
└── 条件执行：控制何时运行

Dockerfile 优化：
├── 多阶段构建
├── 使用 scratch 基础镜像
└── 利用缓存层
```
