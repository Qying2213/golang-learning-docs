# Kubernetes 教程1 - 基础概念

## 1. Kubernetes 简介

### 1.1 什么是 Kubernetes？

Kubernetes（简称 K8s）是一个开源的**容器编排平台**，用于自动化部署、扩展和管理容器化应用。

**名字由来：** Kubernetes 是希腊语"舵手"的意思，K 和 s 之间有 8 个字母，所以简称 K8s。

### 1.2 为什么需要 K8s？

**Docker Compose 的局限：**
- 只能在单机运行
- 没有自动扩缩容
- 没有自动故障恢复
- 不适合生产环境大规模部署

**K8s 解决的问题：**
- **多机部署**：管理多台服务器上的容器
- **自动调度**：智能分配容器到合适的节点
- **自动扩缩容**：根据负载自动增减容器数量
- **自动恢复**：容器挂了自动重启
- **滚动更新**：无停机更新应用
- **服务发现**：自动负载均衡
- **配置管理**：统一管理配置和密钥

### 1.3 K8s 架构

```
+------------------------------------------------------------------+
|                        Kubernetes 集群                            |
|  +------------------------------------------------------------+  |
|  |                    Control Plane（控制平面）                  |  |
|  |  +------------+  +------------+  +----------+  +----------+ |  |
|  |  | API Server |  | Scheduler  |  | Controller|  |  etcd    | |  |
|  |  +------------+  +------------+  | Manager   |  +----------+ |  |
|  |                                  +----------+                |  |
|  +------------------------------------------------------------+  |
|                              ↓                                    |
|  +------------------------------------------------------------+  |
|  |                      Worker Nodes（工作节点）                  |  |
|  |  +------------------+  +------------------+                  |  |
|  |  |     Node 1       |  |     Node 2       |  ...            |  |
|  |  | +-----+ +-----+  |  | +-----+ +-----+  |                  |  |
|  |  | | Pod | | Pod |  |  | | Pod | | Pod |  |                  |  |
|  |  | +-----+ +-----+  |  | +-----+ +-----+  |                  |  |
|  |  | kubelet, kube-proxy                    |                  |  |
|  |  +------------------+  +------------------+                  |  |
|  +------------------------------------------------------------+  |
+------------------------------------------------------------------+
```

**Control Plane 组件：**
| 组件 | 作用 |
|------|------|
| API Server | 集群的入口，所有操作都通过它 |
| Scheduler | 决定 Pod 运行在哪个节点 |
| Controller Manager | 管理各种控制器（副本、节点等） |
| etcd | 分布式键值存储，保存集群状态 |

**Node 组件：**
| 组件 | 作用 |
|------|------|
| kubelet | 管理节点上的 Pod |
| kube-proxy | 网络代理，实现服务发现和负载均衡 |
| Container Runtime | 容器运行时（Docker、containerd） |

## 2. 核心概念

### 2.1 Pod

Pod 是 K8s 最小的部署单元，包含一个或多个容器。

```
+---------------------------+
|           Pod             |
|  +-------+    +-------+   |
|  |Container|  |Container| |
|  | (app)  |   | (sidecar)| |
|  +-------+    +-------+   |
|       共享网络和存储        |
+---------------------------+
```

**特点：**
- 同一 Pod 内的容器共享网络（localhost 互访）
- 同一 Pod 内的容器共享存储卷
- Pod 是临时的，随时可能被销毁重建
- 通常一个 Pod 运行一个主容器

### 2.2 Deployment

Deployment 用于管理 Pod 的副本数量和更新策略。

```yaml
Deployment
    ├── 管理 ReplicaSet
    │       └── 管理 Pod 副本
    ├── 滚动更新
    └── 回滚
```

### 2.3 Service

Service 为一组 Pod 提供稳定的访问入口和负载均衡。

```
              +----------+
              | Service  |
              | (稳定IP)  |
              +----+-----+
                   |
       +-----------+-----------+
       ↓           ↓           ↓
   +-------+   +-------+   +-------+
   | Pod 1 |   | Pod 2 |   | Pod 3 |
   +-------+   +-------+   +-------+
```

**为什么需要 Service？**
- Pod IP 是动态的，重建后会变
- Service 提供稳定的 IP 和 DNS 名称
- 自动负载均衡到后端 Pod

### 2.4 Namespace

Namespace 用于隔离资源，类似于文件夹。

```bash
# 默认命名空间
default          # 默认
kube-system      # K8s 系统组件
kube-public      # 公共资源
```

### 2.5 概念关系图

```
Namespace（命名空间）
    └── Deployment（部署）
            └── ReplicaSet（副本集）
                    └── Pod（容器组）
                            └── Container（容器）

    └── Service（服务）→ 指向 Pod

    └── ConfigMap（配置）→ 被 Pod 使用
    └── Secret（密钥）→ 被 Pod 使用
    └── PersistentVolumeClaim（存储）→ 被 Pod 使用
```

## 3. 安装本地 K8s 环境

### 3.1 使用 Docker Desktop

Docker Desktop 自带 K8s，最简单的方式：

1. 打开 Docker Desktop
2. 进入 Settings → Kubernetes
3. 勾选 "Enable Kubernetes"
4. 点击 "Apply & Restart"
5. 等待安装完成（可能需要几分钟）

### 3.2 使用 Minikube

```bash
# macOS 安装
brew install minikube

# 启动集群
minikube start

# 查看状态
minikube status

# 停止集群
minikube stop

# 删除集群
minikube delete
```

### 3.3 使用 Kind（Kubernetes in Docker）

```bash
# 安装
brew install kind

# 创建集群
kind create cluster --name my-cluster

# 查看集群
kind get clusters

# 删除集群
kind delete cluster --name my-cluster
```

### 3.4 验证安装

```bash
# 查看集群信息
kubectl cluster-info

# 查看节点
kubectl get nodes

# 查看所有 Pod
kubectl get pods -A
```

## 4. kubectl 基础命令

kubectl 是 K8s 的命令行工具，格式：
```bash
kubectl [命令] [资源类型] [资源名] [选项]
```

### 4.1 查看资源

```bash
# 查看 Pod
kubectl get pods
kubectl get pods -o wide          # 显示更多信息
kubectl get pods -A               # 所有命名空间
kubectl get pods -n kube-system   # 指定命名空间

# 查看 Deployment
kubectl get deployments
kubectl get deploy                # 简写

# 查看 Service
kubectl get services
kubectl get svc                   # 简写

# 查看所有资源
kubectl get all

# 查看详细信息
kubectl describe pod <pod-name>
kubectl describe deploy <deploy-name>
```

### 4.2 创建资源

```bash
# 从 YAML 文件创建
kubectl apply -f deployment.yaml

# 创建命名空间
kubectl create namespace dev

# 快速创建 Deployment
kubectl create deployment nginx --image=nginx

# 快速创建 Service
kubectl expose deployment nginx --port=80 --type=NodePort
```

### 4.3 删除资源

```bash
# 删除 Pod
kubectl delete pod <pod-name>

# 删除 Deployment
kubectl delete deployment <deploy-name>

# 从 YAML 文件删除
kubectl delete -f deployment.yaml

# 删除命名空间（会删除其中所有资源）
kubectl delete namespace dev
```

### 4.4 调试命令

```bash
# 查看 Pod 日志
kubectl logs <pod-name>
kubectl logs -f <pod-name>        # 实时日志
kubectl logs <pod-name> -c <container-name>  # 多容器 Pod

# 进入 Pod
kubectl exec -it <pod-name> -- bash
kubectl exec -it <pod-name> -- sh

# 端口转发（本地访问 Pod）
kubectl port-forward <pod-name> 8080:80

# 查看事件
kubectl get events
```

### 4.5 常用简写

| 资源 | 简写 |
|------|------|
| pods | po |
| deployments | deploy |
| services | svc |
| namespaces | ns |
| configmaps | cm |
| secrets | - |
| persistentvolumeclaims | pvc |

## 5. 第一个 K8s 应用

### 5.1 运行 Nginx

```bash
# 创建 Deployment
kubectl create deployment nginx --image=nginx

# 查看 Pod
kubectl get pods

# 暴露服务
kubectl expose deployment nginx --port=80 --type=NodePort

# 查看 Service
kubectl get svc nginx

# 获取访问地址（Minikube）
minikube service nginx --url

# 或者端口转发
kubectl port-forward svc/nginx 8080:80
# 访问 http://localhost:8080
```

### 5.2 使用 YAML 文件

创建 `nginx-deployment.yaml`：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.24
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
  type: NodePort
```

```bash
# 应用配置
kubectl apply -f nginx-deployment.yaml

# 查看资源
kubectl get deploy,pods,svc

# 删除资源
kubectl delete -f nginx-deployment.yaml
```

### 练习4：YAML 部署

创建一个 YAML 文件，部署一个 Redis 应用：
- Deployment 名称：redis
- 副本数：1
- 镜像：redis:7-alpine
- Service 类型：ClusterIP
- 端口：6379

<details>
<summary>点击查看答案</summary>

```yaml
# redis.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: redis
spec:
  replicas: 1
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
      - name: redis
        image: redis:7-alpine
        ports:
        - containerPort: 6379
---
apiVersion: v1
kind: Service
metadata:
  name: redis
spec:
  selector:
    app: redis
  ports:
  - port: 6379
    targetPort: 6379
  type: ClusterIP
```

```bash
kubectl apply -f redis.yaml
kubectl get deploy,pods,svc
```
</details>
