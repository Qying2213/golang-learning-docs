# Kubernetes 教程2 - 核心资源详解

## 1. Deployment 详解

### 1.1 Deployment 的作用

- 管理 Pod 的副本数量
- 滚动更新和回滚
- 自动恢复失败的 Pod

### 1.2 完整的 Deployment YAML

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app                    # Deployment 名称
  namespace: default              # 命名空间
  labels:
    app: my-app
spec:
  replicas: 3                     # 副本数量
  
  selector:                       # 选择器，匹配 Pod
    matchLabels:
      app: my-app
  
  strategy:                       # 更新策略
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1                 # 更新时最多多创建1个Pod
      maxUnavailable: 0           # 更新时最多不可用0个Pod
  
  template:                       # Pod 模板
    metadata:
      labels:
        app: my-app               # 必须与 selector 匹配
    spec:
      containers:
      - name: app
        image: my-app:v1
        ports:
        - containerPort: 8080
        
        resources:                # 资源限制
          requests:
            memory: "64Mi"
            cpu: "100m"
          limits:
            memory: "128Mi"
            cpu: "200m"
        
        env:                      # 环境变量
        - name: APP_ENV
          value: "production"
        
        livenessProbe:            # 存活探针
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 10
        
        readinessProbe:           # 就绪探针
          httpGet:
            path: /ready
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 5
```

### 1.3 更新策略

**RollingUpdate（滚动更新，默认）：**
```yaml
strategy:
  type: RollingUpdate
  rollingUpdate:
    maxSurge: 25%         # 最多多出25%的Pod
    maxUnavailable: 25%   # 最多25%不可用
```

**Recreate（重建）：**
```yaml
strategy:
  type: Recreate          # 先删除所有旧Pod，再创建新Pod
```

### 1.4 滚动更新和回滚

```bash
# 更新镜像
kubectl set image deployment/my-app app=my-app:v2

# 查看更新状态
kubectl rollout status deployment/my-app

# 查看更新历史
kubectl rollout history deployment/my-app

# 回滚到上一版本
kubectl rollout undo deployment/my-app

# 回滚到指定版本
kubectl rollout undo deployment/my-app --to-revision=2

# 暂停更新
kubectl rollout pause deployment/my-app

# 恢复更新
kubectl rollout resume deployment/my-app
```

### 1.5 健康检查

**存活探针（livenessProbe）：** 检测容器是否存活，失败则重启容器

**就绪探针（readinessProbe）：** 检测容器是否准备好接收流量，失败则从 Service 移除

**启动探针（startupProbe）：** 检测容器是否启动完成，适用于启动慢的应用

```yaml
# HTTP 探针
livenessProbe:
  httpGet:
    path: /health
    port: 8080
  initialDelaySeconds: 10   # 启动后等待10秒
  periodSeconds: 10         # 每10秒检查一次
  timeoutSeconds: 5         # 超时时间
  failureThreshold: 3       # 失败3次后重启

# TCP 探针
livenessProbe:
  tcpSocket:
    port: 8080
  initialDelaySeconds: 10
  periodSeconds: 10

# 命令探针
livenessProbe:
  exec:
    command:
    - cat
    - /tmp/healthy
  initialDelaySeconds: 10
  periodSeconds: 10
```

## 2. Service 详解

### 2.1 Service 类型

| 类型 | 说明 | 使用场景 |
|------|------|----------|
| ClusterIP | 集群内部访问（默认） | 内部服务通信 |
| NodePort | 通过节点端口访问 | 开发测试 |
| LoadBalancer | 云厂商负载均衡器 | 生产环境对外服务 |
| ExternalName | 映射外部服务 | 访问集群外服务 |

### 2.2 ClusterIP

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  type: ClusterIP           # 默认类型
  selector:
    app: my-app
  ports:
  - port: 80                # Service 端口
    targetPort: 8080        # Pod 端口
```

```bash
# 集群内访问
curl http://my-service:80
curl http://my-service.default.svc.cluster.local:80
```

### 2.3 NodePort

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  type: NodePort
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
    nodePort: 30080         # 节点端口（30000-32767）
```

```bash
# 通过节点IP访问
curl http://<node-ip>:30080
```

### 2.4 LoadBalancer

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  type: LoadBalancer
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
```

### 2.5 多端口 Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-service
spec:
  selector:
    app: my-app
  ports:
  - name: http
    port: 80
    targetPort: 8080
  - name: https
    port: 443
    targetPort: 8443
```

## 3. ConfigMap - 配置管理

### 3.1 创建 ConfigMap

**方式1：命令行创建**
```bash
# 从字面值创建
kubectl create configmap my-config \
  --from-literal=APP_ENV=production \
  --from-literal=LOG_LEVEL=info

# 从文件创建
kubectl create configmap my-config --from-file=config.yaml

# 从目录创建
kubectl create configmap my-config --from-file=./configs/
```

**方式2：YAML 文件**
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-config
data:
  APP_ENV: production
  LOG_LEVEL: info
  config.yaml: |
    server:
      port: 8080
    database:
      host: localhost
```

### 3.2 使用 ConfigMap

**方式1：环境变量**
```yaml
spec:
  containers:
  - name: app
    image: my-app
    env:
    # 引用单个值
    - name: APP_ENV
      valueFrom:
        configMapKeyRef:
          name: my-config
          key: APP_ENV
    
    # 引用所有值
    envFrom:
    - configMapRef:
        name: my-config
```

**方式2：挂载为文件**
```yaml
spec:
  containers:
  - name: app
    image: my-app
    volumeMounts:
    - name: config-volume
      mountPath: /etc/config
  volumes:
  - name: config-volume
    configMap:
      name: my-config
```

## 4. Secret - 密钥管理

### 4.1 创建 Secret

```bash
# 从字面值创建
kubectl create secret generic my-secret \
  --from-literal=username=admin \
  --from-literal=password=123456

# 从文件创建
kubectl create secret generic my-secret --from-file=./secrets/
```

**YAML 方式（需要 base64 编码）：**
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
type: Opaque
data:
  username: YWRtaW4=        # echo -n 'admin' | base64
  password: MTIzNDU2        # echo -n '123456' | base64
```

**或使用 stringData（自动编码）：**
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: my-secret
type: Opaque
stringData:
  username: admin
  password: "123456"
```

### 4.2 使用 Secret

```yaml
spec:
  containers:
  - name: app
    image: my-app
    env:
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: my-secret
          key: password
    
    # 挂载为文件
    volumeMounts:
    - name: secret-volume
      mountPath: /etc/secrets
      readOnly: true
  volumes:
  - name: secret-volume
    secret:
      secretName: my-secret
```

## 5. PersistentVolume - 持久化存储

### 5.1 存储概念

```
PersistentVolume (PV)     - 集群级别的存储资源
        ↑
        | 绑定
        ↓
PersistentVolumeClaim (PVC) - 用户对存储的请求
        ↑
        | 使用
        ↓
      Pod
```

### 5.2 创建 PV 和 PVC

**PersistentVolume：**
```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce       # 单节点读写
  persistentVolumeReclaimPolicy: Retain
  hostPath:               # 本地测试用
    path: /data/my-pv
```

**PersistentVolumeClaim：**
```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi
```

**在 Pod 中使用：**
```yaml
spec:
  containers:
  - name: app
    image: my-app
    volumeMounts:
    - name: data
      mountPath: /data
  volumes:
  - name: data
    persistentVolumeClaim:
      claimName: my-pvc
```

### 5.3 访问模式

| 模式 | 简写 | 说明 |
|------|------|------|
| ReadWriteOnce | RWO | 单节点读写 |
| ReadOnlyMany | ROX | 多节点只读 |
| ReadWriteMany | RWX | 多节点读写 |

## 6. 实战：部署完整应用

### 6.1 项目结构

```
k8s-demo/
├── namespace.yaml
├── configmap.yaml
├── secret.yaml
├── mysql/
│   ├── deployment.yaml
│   ├── service.yaml
│   └── pvc.yaml
└── app/
    ├── deployment.yaml
    └── service.yaml
```

### 6.2 完整配置

**namespace.yaml：**
```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: demo
```

**configmap.yaml：**
```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
  namespace: demo
data:
  APP_ENV: production
  DB_HOST: mysql
  DB_PORT: "3306"
  DB_NAME: myapp
```

**secret.yaml：**
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: app-secret
  namespace: demo
type: Opaque
stringData:
  DB_USER: root
  DB_PASSWORD: "123456"
  MYSQL_ROOT_PASSWORD: "123456"
```

**mysql/pvc.yaml：**
```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: mysql-pvc
  namespace: demo
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
```

**mysql/deployment.yaml：**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mysql
  namespace: demo
spec:
  replicas: 1
  selector:
    matchLabels:
      app: mysql
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
      - name: mysql
        image: mysql:8.0
        ports:
        - containerPort: 3306
        env:
        - name: MYSQL_ROOT_PASSWORD
          valueFrom:
            secretKeyRef:
              name: app-secret
              key: MYSQL_ROOT_PASSWORD
        - name: MYSQL_DATABASE
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: DB_NAME
        volumeMounts:
        - name: mysql-data
          mountPath: /var/lib/mysql
      volumes:
      - name: mysql-data
        persistentVolumeClaim:
          claimName: mysql-pvc
```

**mysql/service.yaml：**
```yaml
apiVersion: v1
kind: Service
metadata:
  name: mysql
  namespace: demo
spec:
  selector:
    app: mysql
  ports:
  - port: 3306
    targetPort: 3306
  type: ClusterIP
```

**app/deployment.yaml：**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  namespace: demo
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: app
        image: my-go-app:latest
        ports:
        - containerPort: 8080
        envFrom:
        - configMapRef:
            name: app-config
        - secretRef:
            name: app-secret
        resources:
          requests:
            memory: "64Mi"
            cpu: "100m"
          limits:
            memory: "128Mi"
            cpu: "200m"
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 5
```

**app/service.yaml：**
```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
  namespace: demo
spec:
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
  type: NodePort
```

### 6.3 部署命令

```bash
# 创建所有资源
kubectl apply -f namespace.yaml
kubectl apply -f configmap.yaml
kubectl apply -f secret.yaml

kubectl logs -f deployment/my-app -n demo

# 端口转发测试
```

## 8. 本章小结

**核心资源：**
| 资源 | 作用 |
|------|------|
| Deployment | 管理 Pod 副本和更新 |
| Service | 服务发现和负载均衡 |
| ConfigMap | 配置管理 |
| Secret | 密钥管理 |
| PVC | 持久化存储 |

**Service 类型：**
- ClusterIP：集群内部访问
- NodePort：节点端口访问
- LoadBalancer：负载均衡器

**健康检查：**
- livenessProbe：存活探针
- readinessProbe：就绪探针
- startupProbe：启动探针
