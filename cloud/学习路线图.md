# Docker & Kubernetes 学习路线图

## 学习原则

- ⭐⭐⭐ = 必须掌握（工作必用）
- ⭐⭐ = 重要（常用）
- ⭐ = 了解即可（不常用）

---

## 第一阶段：Docker 基础（1-2周）

### 1. 核心概念 ⭐⭐⭐

**必须理解：**
- 镜像 vs 容器的区别
- Docker 架构（Client-Server）
- 为什么需要 Docker

**学习材料：**
- `Docker核心概念-镜像与容器.md`

**掌握标准：**
- 能用一句话解释镜像和容器
- 知道 Docker Daemon 是什么

---

### 2. 基础命令 ⭐⭐⭐

**必须会用：**
```bash
# 镜像操作
docker pull          # 拉取镜像
docker images        # 查看镜像
docker rmi           # 删除镜像

# 容器操作
docker run           # 运行容器（最重要！）
docker ps            # 查看容器
docker stop/start    # 停止/启动
docker rm            # 删除容器
docker logs          # 查看日志
docker exec          # 进入容器
```

**学习材料：**
- `Docker教程1-基础入门.md`

**掌握标准：**
- 能独立运行 nginx、mysql、redis 容器
- 能配置端口映射、环境变量
- 会查看日志和进入容器调试

**练习任务：**
1. 运行一个 nginx，映射到 8080 端口
2. 运行一个 MySQL，设置密码，创建数据库
3. 进入容器，查看配置文件

---

### 3. Dockerfile ⭐⭐⭐

**必须掌握的指令：**
```dockerfile
FROM         # 基础镜像
WORKDIR      # 工作目录
COPY         # 复制文件
RUN          # 执行命令
ENV          # 环境变量
EXPOSE       # 声明端口
CMD          # 启动命令
```

**了解即可：**
```dockerfile
ENTRYPOINT   # 入口点（不常改）
ARG          # 构建参数（偶尔用）
VOLUME       # 数据卷（一般用 -v）
```

**学习材料：**
- `Docker教程2-Dockerfile详解.md`

**掌握标准：**
- 能为 Go/Node/Python 项目写 Dockerfile
- 理解多阶段构建（减小镜像体积）
- 会用 .dockerignore

**练习任务：**
1. 为你的 Go 项目写 Dockerfile
2. 构建镜像并运行
3. 优化镜像大小（多阶段构建）

---

### 4. 数据持久化 ⭐⭐

**必须掌握：**
```bash
# 数据卷（推荐）
docker run -v my-data:/var/lib/mysql mysql

# 绑定挂载（开发常用）
docker run -v $(pwd)/data:/app/data nginx
```

**掌握标准：**
- 知道容器删除后数据会丢失
- 会用 -v 持久化数据
- 理解数据卷和绑定挂载的区别

**练习任务：**
1. 运行 MySQL，用数据卷持久化
2. 删除容器，重新创建，验证数据还在

---

### 5. Docker Compose ⭐⭐⭐

**必须掌握：**
```yaml
version: '3.8'
services:
  web:
    image: nginx
    ports:
      - "8080:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: 123456
    volumes:
      - db-data:/var/lib/mysql
volumes:
  db-data:
```

**常用命令：**
```bash
docker compose up -d      # 启动
docker compose down       # 停止并删除
docker compose ps         # 查看状态
docker compose logs -f    # 查看日志
```

**学习材料：**
- `Docker教程3-Compose编排.md`

**掌握标准：**
- 能写 docker-compose.yml
- 能用 Compose 启动多容器应用（前端+后端+数据库）

**练习任务：**
1. 用 Compose 部署：Go API + MySQL + Redis
2. 配置网络，让容器间能互相访问

---

## 第二阶段：Kubernetes 基础（2-3周）

### 1. 核心概念 ⭐⭐⭐

**必须理解：**
- Pod：最小部署单元
- Deployment：管理 Pod 副本
- Service：服务发现和负载均衡
- Namespace：资源隔离

**不用深究：**
- ReplicaSet（Deployment 会自动管理）
- DaemonSet、StatefulSet（特殊场景）

**学习材料：**
- `K8s教程1-基础概念.md`

**掌握标准：**
- 能画出 Pod、Deployment、Service 的关系图
- 理解为什么需要 Service（Pod IP 会变）

---

### 2. kubectl 命令 ⭐⭐⭐

**必须会用：**
```bash
# 查看资源
kubectl get pods/deploy/svc
kubectl describe pod <name>

# 创建/删除
kubectl apply -f xxx.yaml
kubectl delete -f xxx.yaml

# 调试
kubectl logs <pod>
kubectl exec -it <pod> -- bash
kubectl port-forward <pod> 8080:80

# 扩缩容
kubectl scale deployment <name> --replicas=3
```

**学习材料：**
- `K8s教程1-基础概念.md`

**掌握标准：**
- 能用 kubectl 查看、创建、删除资源
- 会查看日志和进入 Pod 调试

---

### 3. YAML 配置 ⭐⭐⭐

**必须掌握：**

**Deployment：**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: app
        image: my-app:v1
        ports:
        - containerPort: 8080
```

**Service：**
```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
spec:
  selector:
    app: my-app
  ports:
  - port: 80
    targetPort: 8080
  type: ClusterIP  # 或 NodePort、LoadBalancer
```

**学习材料：**
- `K8s教程2-核心资源.md`

**掌握标准：**
- 能写 Deployment 和 Service 的 YAML
- 理解 selector 和 labels 的作用

---

### 4. ConfigMap & Secret ⭐⭐

**必须掌握：**
```yaml
# ConfigMap
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  DB_HOST: mysql
  DB_PORT: "3306"

---
# Secret
apiVersion: v1
kind: Secret
metadata:
  name: app-secret
stringData:
  DB_PASSWORD: "123456"
```

**使用方式：**
```yaml
env:
- name: DB_HOST
  valueFrom:
    configMapKeyRef:
      name: app-config
      key: DB_HOST
```

**掌握标准：**
- 知道配置和密码不要硬编码
- 会用 ConfigMap 和 Secret

---

### 5. 健康检查 ⭐⭐

**必须掌握：**
```yaml
livenessProbe:    # 存活探针：挂了就重启
  httpGet:
    path: /health
    port: 8080
  initialDelaySeconds: 10
  periodSeconds: 10

readinessProbe:   # 就绪探针：没准备好不接收流量
  httpGet:
    path: /ready
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 5
```

**掌握标准：**
- 知道为什么需要健康检查
- 能配置 HTTP 探针

---

### 6. Ingress ⭐⭐

**必须掌握：**
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: my-ingress
spec:
  rules:
  - host: myapp.com
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: api-service
            port:
              number: 80
      - path: /
        pathType: Prefix
        backend:
          service:
            name: web-service
            port:
              number: 80
```

**学习材料：**
- `K8s教程3-Ingress与实战.md`

**掌握标准：**
- 理解 Ingress 是统一入口
- 能配置域名和路径路由

## 学习建议

1. **边学边练**：每学一个知识点，立即动手实践
2. **先会用再深究**：先把命令用起来，再研究原理
3. **重点突破**：把 ⭐⭐⭐ 的内容练熟，⭐ 的内容了解即可
4. **做项目巩固**：学完基础后，一定要做实战项目
5. **遇到问题先查日志**：`docker logs` 和 `kubectl logs` 是最好的老师

## 常见问题

**Q: Docker 和 K8s 哪个更重要？**
A: 都重要。Docker 是基础，K8s 是进阶。先学 Docker，再学 K8s。

**Q: 需要学习 Docker Swarm 吗？**
A: 不需要。Swarm 已经被 K8s 取代，直接学 K8s。

**Q: 需要深入学习网络和存储吗？**
A: 不需要。会用就行，深入的内容是运维工程师关注的。

**Q: 学完能找工作吗？**
A: 学完基础 + 做几个项目，可以应对初级岗位。中高级岗位还需要学习 CI/CD、监控、服务网格等。

**Q: 学习资料太多，看不完怎么办？**
A: 按照这个路线图，只学 ⭐⭐⭐ 和 ⭐⭐ 的内容，其他的用到再学。
