# Docker 核心概念 - 镜像与容器

## 1. 镜像和容器的区别

### 1.1 一句话理解

- **镜像（Image）**= 安装包 / 类（Class）/ 模板
- **容器（Container）**= 运行中的程序 / 对象（Object）/ 实例

### 1.2 生活类比

| 概念 | 类比1 | 类比2 | 类比3 |
|------|-------|-------|-------|
| 镜像 | 蛋糕模具 | 光盘/ISO文件 | 菜谱 |
| 容器 | 做出来的蛋糕 | 安装后运行的程序 | 做出来的菜 |

**关键区别：**
- 一个模具可以做出很多蛋糕
- 一个镜像可以创建很多容器

### 1.3 技术角度

```
+------------------+          +------------------+
|      镜像        |          |      容器        |
+------------------+          +------------------+
| - 只读           |   运行   | - 可读写         |
| - 静态文件       |  ----→  | - 运行中的进程    |
| - 分层存储       |          | - 有自己的文件系统 |
| - 可以分享       |          | - 有自己的网络    |
+------------------+          +------------------+
```

**镜像特点：**
- ✅ 只读，不可修改
- ✅ 分层结构，共享基础层
- ✅ 可以推送到仓库分享
- ✅ 一次构建，到处运行

**容器特点：**
- ✅ 可读写（在镜像上加一层可写层）
- ✅ 有独立的进程空间
- ✅ 有独立的网络和端口
- ✅ 生命周期：创建 → 运行 → 停止 → 删除

### 1.4 代码类比（Go 语言）

```go
// 镜像 = 类型定义
type Nginx struct {
    Config string
    Port   int
}

// 容器 = 实例化的对象
nginx1 := Nginx{Config: "site1.conf", Port: 8080}  // 容器1
nginx2 := Nginx{Config: "site2.conf", Port: 8081}  // 容器2
nginx3 := Nginx{Config: "site3.conf", Port: 8082}  // 容器3

// 一个类型（镜像）可以创建多个实例（容器）
```

## 2. 镜像的分层结构

### 2.1 镜像是怎么构成的？

镜像由多个**只读层**叠加而成：

```
+------------------------+
|    应用代码层          |  ← 你的代码
+------------------------+
|    依赖安装层          |  ← npm install / go mod
+------------------------+
|    系统工具层          |  ← apt-get install
+------------------------+
|    基础镜像层          |  ← ubuntu / alpine
+------------------------+
```

### 2.2 为什么要分层？

**节省空间和时间！**

```
镜像A (nginx)           镜像B (你的应用)
+-------------+         +-------------+
| nginx 配置  |         | 你的代码    |
+-------------+         +-------------+
| 基础镜像    |  ←共享→ | 基础镜像    |
+-------------+         +-------------+

两个镜像共享同一个基础层，不会重复存储
```

## 3. 容器的生命周期

```
                    docker create
                         ↓
+----------+      +------------+      docker start      +----------+
|  镜像    | --→  |  已创建    | ------------------→   |  运行中  |
|  Image   |      |  Created   |                       |  Running |
+----------+      +------------+                       +----------+
                         ↑                                  |
                         |         docker stop              ↓
                         +---------------------------- +----------+
                                                       |  已停止  |
                                   docker rm           |  Stopped |
                              +--------------------→   +----------+
                              |                             |
                         +----------+                       |
                         |  已删除  | ←---------------------+
                         |  Deleted |      docker rm
                         +----------+
```

### 3.1 生命周期命令

```bash
# 创建容器（不启动）
docker create --name my-nginx nginx

# 启动容器
docker start my-nginx

# 创建并启动（最常用）
docker run -d --name my-nginx nginx

# 停止容器
docker stop my-nginx

# 重启容器
docker restart my-nginx

# 删除容器（必须先停止）
docker rm my-nginx

# 强制删除运行中的容器
docker rm -f my-nginx
```

## 4. Docker 完整启动流程

### 4.1 当你执行 `docker run nginx` 时发生了什么？

### 4.2 详细步骤解释

**步骤1：Docker 客户端解析命令**
```bash
docker run -d -p 8080:80 --name web nginx
#          │   │         │          │
#          │   │         │          └── 镜像名
#          │   │         └── 容器名
#          │   └── 端口映射（宿主机:容器）
#          └── 后台运行
```

**步骤2：检查本地镜像**
```bash
# Docker 会检查本地是否有 nginx 镜像
docker images nginx
```

**步骤3：拉取镜像（如果本地没有）**
```bash
# 从 Docker Hub 下载
docker pull nginx
```

**步骤4：创建容器**
- 基于镜像创建一个可写层
- 分配容器 ID
- 准备文件系统

**步骤5：配置容器**
- 分配 IP 地址
- 设置端口映射（8080 → 80）
- 挂载数据卷（如果有 -v 参数）
- 设置环境变量（如果有 -e 参数）

**步骤6：启动容器**
- 执行镜像中定义的启动命令
- nginx 镜像的启动命令是：`nginx -g 'daemon off;'`

### 4.3 实际演示

```bash
# 1. 确保没有 nginx 镜像
docker rmi nginx 2>/dev/null

# 2. 运行 nginx（会自动拉取）
docker run -d -p 8080:80 --name web nginx

# 输出：
# Unable to find image 'nginx:latest' locally     ← 本地没有
# latest: Pulling from library/nginx              ← 开始拉取
# ...
# Status: Downloaded newer image for nginx:latest ← 拉取完成
# abc123def456...                                 ← 容器 ID

# 3. 查看运行中的容器
docker ps

# 4. 访问
curl http://localhost:8080

# 5. 查看容器日志
docker logs web

# 6. 停止并删除
docker stop web && docker rm web
```

## 5. 镜像和容器的存储位置

### 5.1 它们存在哪里？

```bash
# Docker 数据目录（Linux）
/var/lib/docker/
├── image/          # 镜像元数据
├── containers/     # 容器数据
├── volumes/        # 数据卷
├── network/        # 网络配置
└── overlay2/       # 镜像层文件（分层存储）

# macOS 上 Docker Desktop 使用虚拟机
# 数据在 ~/Library/Containers/com.docker.docker/Data/
```

### 5.2 查看占用空间

```bash
# 查看 Docker 磁盘使用
docker system df

# 输出示例：
TYPE            TOTAL     ACTIVE    SIZE      RECLAIMABLE
Images          5         2         1.2GB     800MB (66%)
Containers      3         1         50MB      30MB (60%)
Local Volumes   2         1         500MB     200MB (40%)
Build Cache     10        0         300MB     300MB (100%)

# 清理未使用的资源

```

## 6. 常见问题

### Q1: 容器删除后数据会丢失吗？

**会！** 容器的可写层会随容器删除而消失。

**解决方案：** 使用数据卷持久化
```bash
docker run -v /host/data:/container/data nginx
```

### Q2: 一个镜像能运行多个容器吗？

**能！** 这是 Docker 的核心特性。
```bash
docker run -d -p 8081:80 --name web1 nginx
docker run -d -p 8082:80 --name web2 nginx
docker run -d -p 8083:80 --name web3 nginx
# 三个容器，同一个镜像
```

### Q3: 容器之间会互相影响吗？

**不会！** 每个容器有独立的：
- 文件系统
- 网络空间
- 进程空间

## 7. 总结

| 对比项 | 镜像 Image | 容器 Container |
|--------|-----------|----------------|
| 状态 | 静态、只读 | 动态、可读写 |
| 类比 | 类/模板 | 对象/实例 |
| 存储 | 分层存储 | 镜像 + 可写层 |
| 数量关系 | 1 个镜像 | 可创建 N 个容器 |
| 生命周期 | 长期存在 | 可随时创建/销毁 |
| 分享 | 可推送到仓库 | 不能直接分享 |

**记住这个公式：**
```
镜像 + 运行 = 容器
Image + Run = Container
```
