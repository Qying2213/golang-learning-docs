# 第一部分：Minikube 命令

## 1. 集群生命周期

```bash
# 启动集群（首次或重新启动）
minikube start

# 启动时指定资源
minikube start --cpus=4 --memory=8192

# 启动时使用国内镜像加速
minikube start --image-mirror-country='cn'

# 启动时指定 Kubernetes 版本
minikube start --kubernetes-version=v1.28.0

# 启动时指定驱动（docker/virtualbox/hyperkit）
minikube start --driver=docker

# 停止集群（保留数据）
minikube stop

# 删除集群（清空所有数据）
minikube delete

# 删除所有集群
minikube delete --all

# 暂停集群（释放 CPU，保留内存）
minikube pause

# 恢复暂停的集群
minikube unpause
```

## 2. 集群状态查看

```bash
# 查看集群状态
minikube status

# 查看集群 IP 地址
minikube ip

# 查看 Kubernetes 版本
minikube kubectl -- version

# 查看集群日志
minikube logs

# 查看集群日志（实时跟踪）
minikube logs -f

# 查看集群配置
minikube config view
```

## 3. 插件管理

```bash
# 查看所有插件
minikube addons list

# 启用插件
minikube addons enable dashboard
minikube addons enable ingress
minikube addons enable metrics-server
minikube addons enable storage-provisioner

# 禁用插件
minikube addons disable dashboard
```

## 4. 访问服务

```bash
# 打开 Dashboard
minikube dashboard

# 获取服务 URL
minikube service <service-name> --url

# 在浏览器中打开服务
minikube service <service-name>

# 指定命名空间
minikube service <service-name> -n <namespace>

# 开启隧道（用于 LoadBalancer 类型）
minikube tunnel
```

## 5. SSH 和 Docker

```bash
# SSH 登录到 Minikube 虚拟机
minikube ssh

# 在虚拟机内执行命令
minikube ssh -- docker ps
minikube ssh -- ls /var/log

# 配置本地 Docker 使用 Minikube 的 Docker
eval $(minikube docker-env)

# 取消上述配置
eval $(minikube docker-env -u)
```

## 6. 多集群管理

```bash
# 创建新的集群 profile
minikube start -p my-cluster

# 切换 profile
minikube profile my-cluster

# 查看所有 profile
minikube profile list

# 删除指定 profile
minikube delete -p my-cluster
```

## 7. 镜像管理

```bash
# 将本地镜像加载到 Minikube
minikube image load my-image:tag

# 在 Minikube 中构建镜像
minikube image build -t my-image:tag .

# 查看 Minikube 中的镜像
minikube image ls
```

## 4. 创建资源

### 命令式创建（快速测试）

```bash
# 创建命名空间
kubectl create namespace <name>
kubectl create ns <name>

# 快速运行 Pod
kubectl run nginx --image=nginx:alpine
kubectl run nginx --image=nginx:alpine --port=80
kubectl run nginx --image=nginx:alpine --labels="app=web,env=dev"

# 创建 Deployment
kubectl create deployment nginx --image=nginx:alpine
kubectl create deployment nginx --image=nginx:alpine --replicas=3

# 创建 Service（暴露 Deployment）
kubectl expose deployment nginx --port=80 --type=ClusterIP
kubectl expose deployment nginx --port=80 --type=NodePort
kubectl expose deployment nginx --port=80 --type=LoadBalancer
kubectl expose deployment nginx --port=80 --target-port=8080

# 创建 Service（直接创建）
kubectl create service clusterip nginx --tcp=80:80
kubectl create service nodeport nginx --tcp=80:80

# 创建 ConfigMap
kubectl create configmap <name> --from-literal=key1=value1
kubectl create configmap <name> --from-literal=key1=value1 --from-literal=key2=value2
kubectl create configmap <name> --from-file=config.txt
kubectl create configmap <name> --from-file=./configs/
kubectl create configmap <name> --from-env-file=.env

# 创建 Secret
kubectl create secret generic <name> --from-literal=username=admin --from-literal=password=123456
kubectl create secret generic <name> --from-file=./secrets/
kubectl create secret docker-registry <name> --docker-server=<server> --docker-username=<user> --docker-password=<pass>
kubectl create secret tls <name> --cert=cert.pem --key=key.pem

# 创建 Job
kubectl create job <name> --image=busybox -- echo "Hello"

# 创建 CronJob
kubectl create cronjob <name> --image=busybox --schedule="*/5 * * * *" -- echo "Hello"
```

### 声明式创建（生产推荐）

```bash
# 应用 YAML 文件
kubectl apply -f <file.yaml>

# 应用多个文件
kubectl apply -f file1.yaml -f file2.yaml

# 应用目录下所有 YAML
kubectl apply -f ./manifests/

# 递归应用
kubectl apply -f ./manifests/ -R

# 应用远程 YAML
kubectl apply -f https://example.com/manifest.yaml

# 预览变更（不实际执行）
kubectl diff -f <file.yaml>
```

### 生成 YAML 模板

```bash
# 生成 Pod YAML
kubectl run nginx --image=nginx --dry-run=client -o yaml > pod.yaml

# 生成 Deployment YAML
kubectl create deployment nginx --image=nginx --dry-run=client -o yaml > deploy.yaml

# 生成 Service YAML
kubectl expose deployment nginx --port=80 --dry-run=client -o yaml > svc.yaml

# 导出现有资源为 YAML
kubectl get deployment nginx -o yaml > deploy.yaml
kubectl get pod nginx -o yaml > pod.yaml
```

## 5. 编辑资源

```bash
# 直接编辑（打开编辑器）
kubectl edit deployment <name>
kubectl edit pod <name>
kubectl edit configmap <name>

# 指定编辑器
KUBE_EDITOR="code --wait" kubectl edit deployment <name>
KUBE_EDITOR="vim" kubectl edit deployment <name>

# 更新镜像
kubectl set image deployment/<name> <container>=<image>:<tag>
kubectl set image deployment/nginx nginx=nginx:1.19

# 更新资源限制
kubectl set resources deployment/<name> -c=<container> --limits=cpu=200m,memory=512Mi
kubectl set resources deployment/<name> -c=<container> --requests=cpu=100m,memory=256Mi

# 更新环境变量
kubectl set env deployment/<name> KEY=value
kubectl set env deployment/<name> KEY-              # 删除环境变量

# 更新副本数
kubectl scale deployment/<name> --replicas=5

# 自动扩缩容
kubectl autoscale deployment/<name> --min=2 --max=10 --cpu-percent=80

# 添加/修改标签
kubectl label pod <name> key=value
kubectl label pod <name> key=newvalue --overwrite
kubectl label pod <name> key-                       # 删除标签

# 添加/修改注解
kubectl annotate pod <name> key=value
kubectl annotate pod <name> key-                    # 删除注解

# 打补丁
kubectl patch deployment <name> -p '{"spec":{"replicas":3}}'
kubectl patch deployment <name> --type=json -p='[{"op":"replace","path":"/spec/replicas","value":5}]'
```

## 6. 滚动更新和回滚

```bash
# 查看更新状态

# 查看更新历史
kubectl rollout history deployment/<name>

# 查看特定版本详情
kubectl rollout history deployment/<name> --revision=2

# 回滚到上一版本
kubectl rollout undo deployment/<name>

# 回滚到指定版本
kubectl rollout undo deployment/<name> --to-revision=2

# 暂停更新
kubectl rollout pause deployment/<name>

# 恢复更新
kubectl rollout resume deployment/<name>

# 重启（触发滚动更新）
kubectl rollout restart deployment/<name>
```

## 9. 执行命令和交互

## 10. 端口转发

```bash

# Deployment 端口转发

kubectl port-forward --address 0.0.0.0 pod/nginx 8080:80

# 后台运行
kubectl port-forward pod/nginx 8080:80 &
```

# 第五部分：故障排查流程

## 1. Pod 问题排查

```bash
# 第一步：查看 Pod 状态
kubectl get pods -o wide

# 第二步：查看详细信息和事件
kubectl describe pod <name>

# 第三步：查看日志
kubectl logs <name>
kubectl logs <name> --previous    # 如果重启过

# 第四步：进入容器调试
kubectl exec -it <name> -- sh
```

## 2. 常见状态及解决方案

| 状态             | 含义         | 排查方向                         |
| :--------------- | :----------- | :------------------------------- |
| Pending          | 等待调度     | 资源不足？节点选择器不匹配？     |
| ImagePullBackOff | 拉取镜像失败 | 镜像名对吗？网络通吗？需要认证？ |
| CrashLoopBackOff | 容器反复崩溃 | 查看日志 kubectl logs --previous |
| Error            | 容器启动错误 | 命令或参数错误？                 |
| Evicted          | 被驱逐       | 节点资源不足                     |
| OOMKilled        | 内存不足被杀 | 增加内存限制                     |

## 3. Service 问题排查

```bash
# 检查 Service 是否有 Endpoints
kubectl get endpoints <service-name>

# 检查 Service 配置
kubectl describe service <service-name>

# 测试 DNS 解析
kubectl run debug --image=busybox --rm -it -- nslookup <service-name>

# 测试连接
kubectl run debug --image=busybox --rm -it -- wget -qO- http://<service-name>
```
