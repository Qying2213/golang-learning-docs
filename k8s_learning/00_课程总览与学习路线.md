**学完本课程，你将能够：**

- ✅ 理解 K8s 的设计理念和核心架构

- ✅ 独立完成应用的容器化部署
- ✅ 掌握 K8s 的网络、存储、配置管理
- ✅ 具备生产环境故障排查能力

## 🎯 学习建议

### 学习顺序

```
01 环境搭建 → 02 架构理解 → 03 Pod → 04 Deployment → 05 Service → 06 配置 → 07 存储
                                           ↓
                                    08 命令速查 (随时翻阅)
```

### 学习方法

1. **边看边敲**: 每个章节都有实战命令，请在终端中跟着执行
2. **先模仿后理解**: 先跑通流程，再回头理解原理
3. **善用 `kubectl describe`**: 遇到问题先 describe 看 Events
4. **多破坏多修复**: 故意搞坏集群，然后修复它，这是最好的学习方式

### 前置知识

- 基本的 Linux 命令操作
- 了解 Docker 基础概念（镜像、容器）
- 会使用终端/命令行

## 💡 核心概念速览

在开始学习前，先对 K8s 的核心对象有一个整体印象：

```
┌─────────────────────────────────────────────────────────────┐
│                      Kubernetes 集群                         │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                    Namespace (命名空间)                │  │
│  │  ┌─────────────┐   ┌─────────────┐   ┌────────────┐  │  │
│  │  │ Deployment  │   │   Service   │   │  ConfigMap │  │  │
│  │  │  (部署控制)  │   │  (服务入口)  │   │  (配置)    │  │  │
│  │  └──────┬──────┘   └──────┬──────┘   └────────────┘  │  │
│  │         │                 │                           │  │
│  │         ▼                 ▼                           │  │
│  │  ┌─────────────┐   ┌─────────────┐                   │  │
│  │  │ ReplicaSet  │   │    Pod 1    │                   │  │
│  │  │  (副本集)    │◀──│    Pod 2    │                   │  │
│  │  └─────────────┘   │    Pod 3    │                   │  │
│  │                    └─────────────┘                   │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

**一句话理解**:

- **Pod**: 最小单位，里面跑容器
- **Deployment**: 管理 Pod 的数量和更新
- **Service**: 给 Pod 一个固定的访问入口
- **ConfigMap/Secret**: 把配置从代码中分离出来
- **PV/PVC**: 给 Pod 提供持久化存储
