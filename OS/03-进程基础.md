## 1. 什么是进程？

### 1.1 程序 vs 进程

```
程序（Program）：
- 静态的代码文件，存储在磁盘上
- 就像一份菜谱，写在纸上

进程（Process）：
- 程序的一次执行实例，运行在内存中
- 就像厨师按照菜谱做菜的过程

关系：
- 一个程序可以有多个进程（多次运行）
- 一个进程只能对应一个程序
```

**实际例子：**
```bash
# 同一个程序启动多个进程
./myapp &    # 进程 1，PID=1234
./myapp &    # 进程 2，PID=1235
./myapp &    # 进程 3，PID=1236

# 查看
ps aux | grep myapp
# 会看到三个不同的进程
```

### 1.3 Go 程序的内存布局实例

```go
package main

import (
    "fmt"
    "runtime"
    "unsafe"
)

// 全局变量 - 数据段
var globalInitVar = 100      // 已初始化，在数据段
var globalUninitVar int      // 未初始化，在 BSS 段

// 常量 - 代码段（只读）
const constVar = "hello"

func main() {
    // 局部变量 - 栈
    localVar := 200
    
    // 指针指向的数据 - 可能在栈或堆（取决于逃逸分析）
    ptr := new(int)
    *ptr = 300
    
    // 切片底层数组 - 堆
    slice := make([]int, 1000)
    
    // map - 堆
    m := make(map[string]int)
    m["key"] = 1
    
    // 打印地址，观察内存布局
    fmt.Printf("全局变量(已初始化) globalInitVar 地址: %p\n", &globalInitVar)
    fmt.Printf("全局变量(未初始化) globalUninitVar 地址: %p\n", &globalUninitVar)
    fmt.Printf("局部变量 localVar 地址: %p\n", &localVar)
    fmt.Printf("new 分配的 ptr 地址: %p\n", ptr)
    fmt.Printf("slice 底层数组地址: %p\n", &slice[0])
    
    // 查看内存统计
    var m runtime.MemStats
    runtime.ReadMemStats(&m)
    fmt.Printf("\n=== 内存统计 ===\n")
    fmt.Printf("堆分配: %.2f KB\n", float64(m.HeapAlloc)/1024)
    fmt.Printf("栈使用: %.2f KB\n", float64(m.StackInuse)/1024)
    fmt.Printf("堆对象数: %d\n", m.HeapObjects)
}
```

### 🎯 练习题 1.1：内存布局

**问题：** 以下代码中，各变量分别存储在哪个内存区域？

```go
package main

var config = "production"  // A
var count int              // B

func process() {
    data := make([]byte, 1024)  // C
    temp := 100                  // D
    result := &temp              // E
}
```

<details>
<summary>点击查看答案</summary>

- **A (config)**: 数据段 - 已初始化的全局变量
- **B (count)**: BSS 段 - 未初始化的全局变量（零值）
- **C (data)**: 切片头在栈上，底层数组在堆上（make 分配）
- **D (temp)**: 栈 - 局部变量
- **E (result)**: 栈 - 指针本身在栈上，但因为取了 temp 的地址，temp 可能逃逸到堆

**验证方法：**
```bash
go build -gcflags="-m" main.go
# 查看逃逸分析结果
```
</details>

## 2. 进程控制块（PCB）⭐⭐⭐

### 2.1 PCB 详解

PCB（Process Control Block）是操作系统用来管理进程的核心数据结构，相当于进程的"身份证"。

```
┌─────────────────────────────────────────────────────────────┐
│                    PCB 详细内容                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【进程标识信息】                                            │
│  ├─ PID：进程唯一标识符                                     │
│  ├─ PPID：父进程 ID                                         │
│  ├─ UID：用户 ID（谁启动的）                                │
│  ├─ GID：组 ID                                              │
│  └─ 进程组 ID、会话 ID                                      │
│                                                              │
│  【处理器状态信息】（上下文切换时保存/恢复）                 │
│  ├─ 程序计数器（PC）：下一条要执行的指令地址                │
│  ├─ 通用寄存器：计算中间结果                                │
│  ├─ 栈指针（SP）：当前栈顶位置                              │
│  ├─ 状态寄存器：CPU 状态标志                                │
│  └─ 浮点寄存器                                              │
│                                                              │
│  【进程调度信息】                                            │
│  ├─ 进程状态：运行/就绪/阻塞/...                            │
│  ├─ 优先级：调度优先级                                      │
│  ├─ 调度策略：FIFO/RR/CFS 等                                │
│  ├─ 时间片：剩余可运行时间                                  │
│  └─ 等待时间、CPU 使用时间统计                              │
│                                                              │
│  【内存管理信息】                                            │
│  ├─ 页表基址：虚拟地址到物理地址的映射                      │
│  ├─ 代码段、数据段、堆、栈的起止地址                        │
│  └─ 内存使用量统计                                          │
│                                                              │
│  【I/O 和文件信息】                                          │
│  ├─ 打开的文件描述符表                                      │
│  ├─ 当前工作目录                                            │
│  └─ 根目录                                                  │
│                                                              │
│  【信号相关】                                                │
│  ├─ 待处理信号                                              │
│  ├─ 信号掩码                                                │
│  └─ 信号处理函数                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 Linux 中查看进程信息

```bash

# root  1234 0.5  1.2  12345 6789 ?    Sl   10:00 0:05 ./myapp

# 各字段含义：
# USER  - 进程所有者
# PID   - 进程 ID

# RSS   - 常驻内存大小（KB）- 实际占用的物理内存
# TTY   - 终端

# START - 启动时间
# TIME  - 累计 CPU 时间
# COMMAND - 命令
```

### 2.3 Go 中获取进程信息

```go
package main

import (
    "fmt"
    "os"
    "os/user"
    "runtime"
    "syscall"
)

func main() {
    // ========== 进程标识信息 ==========
    fmt.Println("=== 进程标识 ===")
    fmt.Println("PID:", os.Getpid())
    fmt.Println("PPID:", os.Getppid())
    fmt.Println("UID:", os.Getuid())
    fmt.Println("GID:", os.Getgid())
    fmt.Println("EUID:", os.Geteuid())  // 有效用户 ID
    fmt.Println("EGID:", os.Getegid())  // 有效组 ID
    
    // 获取用户名
    u, _ := user.Current()
    fmt.Println("用户名:", u.Username)
    fmt.Println("用户主目录:", u.HomeDir)
    
    // ========== 进程组和会话 ==========
    fmt.Println("\n=== 进程组 ===")
    pgid, _ := syscall.Getpgid(0)
    fmt.Println("进程组 ID:", pgid)
    sid, _ := syscall.Getsid(0)
    fmt.Println("会话 ID:", sid)
    
    // ========== 运行时信息 ==========
    fmt.Println("\n=== 运行时信息 ===")
    fmt.Println("Goroutine 数量:", runtime.NumGoroutine())
    fmt.Println("CPU 核心数:", runtime.NumCPU())
    fmt.Println("GOMAXPROCS:", runtime.GOMAXPROCS(0))
    fmt.Println("Go 版本:", runtime.Version())
    fmt.Println("操作系统:", runtime.GOOS)
    fmt.Println("架构:", runtime.GOARCH)
    
    // ========== 工作目录 ==========
    fmt.Println("\n=== 工作目录 ===")
    wd, _ := os.Getwd()
    fmt.Println("当前工作目录:", wd)
    
    // ========== 环境变量 ==========
    fmt.Println("\n=== 环境变量 ===")
    fmt.Println("PATH:", os.Getenv("PATH"))
    fmt.Println("HOME:", os.Getenv("HOME"))
    fmt.Println("GOPATH:", os.Getenv("GOPATH"))
    
    // ========== 内存统计 ==========
    fmt.Println("\n=== 内存统计 ===")
    var m runtime.MemStats
    runtime.ReadMemStats(&m)
    fmt.Printf("Alloc: %.2f MB\n", float64(m.Alloc)/1024/1024)
    fmt.Printf("TotalAlloc: %.2f MB\n", float64(m.TotalAlloc)/1024/1024)
    fmt.Printf("Sys: %.2f MB\n", float64(m.Sys)/1024/1024)
    fmt.Printf("NumGC: %d\n", m.NumGC)
}
```

## 3. 进程状态 ⭐⭐⭐

### 3.1 进程状态模型


**五状态模型：**

```
                         ┌──────────────┐
              创建进程 ──→│    新建      │
                         │   (New)      │
                         └──────┬───────┘
                                │
                                │ 提交（admit）
                                ↓
    ┌──────────────┐     ┌──────────────┐
    │    就绪      │←────│    运行      │
    │  (Ready)     │ 时间│  (Running)   │
    │              │ 片完│              │
    │  等待 CPU    │─────→│  正在执行    │
    └──────┬───────┘调度  └──────┬───────┘
           ↑                     │
           │                     │ I/O 请求
           │                     │ 等待事件
           │                     ↓
           │              ┌──────────────┐
           │              │    阻塞      │
           └──────────────│  (Blocked)   │
              I/O 完成    │  等待 I/O    │
              事件发生    └──────────────┘
                                │
                                │ 终止
                                ↓
                         ┌──────────────┐
                         │    终止      │
                         │ (Terminated) │
                         └──────────────┘
```

**状态说明：**

| 状态 | 说明 | 触发条件 |
|------|------|---------|
| 新建 (New) | 进程刚被创建 | fork() 调用 |
| 就绪 (Ready) | 等待 CPU 调度 | 时间片用完、I/O 完成 |
| 运行 (Running) | 正在 CPU 上执行 | 被调度器选中 |
| 阻塞 (Blocked) | 等待 I/O 或事件 | 发起 I/O 请求、等待锁 |
| 终止 (Terminated) | 执行完毕 | main 返回、exit() |

### 3.2 Linux 进程状态

```bash
# 查看进程状态
ps aux

# STAT 列的含义：
# R - Running/Runnable（运行/可运行）
# S - Sleeping（可中断睡眠，等待事件）
# D - Disk sleep（不可中断睡眠，等待 I/O）
# T - Stopped（停止，收到 SIGSTOP）
# t - Tracing stop（被调试器停止）
# Z - Zombie（僵尸，已终止但未被回收）
# X - Dead（死亡，不会看到）

# 附加标志：
# < - 高优先级
# N - 低优先级
# L - 有页面锁定在内存
# s - 会话领导者
# l - 多线程
# + - 前台进程组
```

**状态详解：**

```
┌─────────────────────────────────────────────────────────────┐
│                    Linux 进程状态详解                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  R (Running)：                                               │
│  - 正在 CPU 上运行，或在运行队列中等待                       │
│  - 这是唯一真正"运行"的状态                                 │
│                                                              │
│  S (Sleeping - Interruptible)：                              │
│  - 可中断睡眠，等待某个条件                                  │
│  - 可以被信号唤醒                                            │
│  - 最常见的状态（等待用户输入、网络数据等）                  │
│                                                              │
│  D (Disk Sleep - Uninterruptible)：                          │
│  - 不可中断睡眠，通常在等待磁盘 I/O                          │
│  - 不能被信号中断（kill -9 也杀不死）                        │
│  - 如果大量进程处于 D 状态，说明 I/O 有问题                  │
│                                                              │
│  T (Stopped)：                                               │
│  - 进程被暂停                                                │
│  - 收到 SIGSTOP 或 SIGTSTP (Ctrl+Z)                         │
│  - 可以用 SIGCONT 恢复                                       │
│                                                              │
│  Z (Zombie)：                                                │
│  - 进程已终止，但父进程还没有调用 wait() 回收                │
│  - 只占用 PCB，不占用其他资源                                │
│  - 大量僵尸进程说明父进程有 bug                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 Go 程序的状态变化

```go
package main

import (
    "fmt"
    "net/http"
    "os"
    "time"
)

func main() {
    fmt.Println("PID:", os.Getpid())
    
    // 1. 运行状态 (R)：CPU 密集计算
    fmt.Println("\n1. CPU 密集计算（状态应为 R）...")
    sum := 0
    for i := 0; i < 100000000; i++ {
        sum += i
    }
    
    // 2. 睡眠状态 (S)：等待 I/O
    fmt.Println("\n2. 等待网络 I/O（状态应为 S）...")
    go func() {
        resp, _ := http.Get("https://example.com")
        if resp != nil {
            resp.Body.Close()
        }
    }()
    
    // 3. 睡眠状态 (S)：time.Sleep
    fmt.Println("\n3. Sleep（状态应为 S）...")
    time.Sleep(2 * time.Second)
    
    // 4. 睡眠状态 (S)：等待用户输入
    fmt.Println("\n4. 等待输入（状态应为 S）...")
    fmt.Println("按回车继续...")
    fmt.Scanln()
    
    fmt.Println("程序结束")
}
```

### 🎯 练习题 3.1：进程状态

1. Go 程序正在执行 `for` 循环计算
2. Go 程序执行 `time.Sleep(time.Second)`
3. Go 程序执行 `http.Get("https://example.com")`

<details>
<summary>点击查看答案</summary>

1. **R (Running)** - CPU 密集计算，进程在运行或等待 CPU

## 4. 进程的创建 ⭐⭐⭐

### 4.1 fork() 系统调用

在 Unix/Linux 中，创建新进程使用 `fork()` 系统调用：

```
fork() 的工作原理：

父进程                           fork()
   │                               │
   │                               ↓
   │                    ┌─────────────────────┐
   │                    │   复制父进程的：     │
   │                    │   - 代码段（共享）   │
   │                    │   - 数据段（复制）   │
   │                    │   - 堆（复制）       │
   │                    │   - 栈（复制）       │
   │                    │   - PCB（复制）      │
   │                    │   - 文件描述符       │
   │                    └─────────────────────┘
   │                          ↙        ↘
   ↓                         ↓          ↓
父进程继续              子进程开始
fork() 返回子进程 PID   fork() 返回 0
```

**写时复制（Copy-On-Write, COW）：**
```
实际上，fork() 并不会立即复制所有内存，而是使用 COW 技术：

1. fork() 后，父子进程共享相同的物理内存页
2. 只有当某个进程要修改内存时，才会复制那一页
3. 这样可以大大减少 fork() 的开销

┌─────────────────────────────────────────────────────────────┐
│  fork() 后：                                                 │
│                                                              │
│  父进程页表 ──┐                                              │
│              ├──→ 物理内存页（只读）                        │
│  子进程页表 ──┘                                              │
│                                                              │
│  写入时：                                                    │
│                                                              │
│  父进程页表 ────→ 物理内存页 A（可写）                       │
│  子进程页表 ────→ 物理内存页 B（复制，可写）                 │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 实际应用：执行系统命令的工具函数

```go
package main

import (
    "bytes"
    "context"
    "fmt"
    "os/exec"
    "time"
)

// CommandResult 命令执行结果
type CommandResult struct {
    Stdout   string
    Stderr   string
    ExitCode int
    Duration time.Duration
    Err      error
}

// RunCommand 执行命令（带超时）
func RunCommand(timeout time.Duration, name string, args ...string) CommandResult {
    start := time.Now()
    
    ctx, cancel := context.WithTimeout(context.Background(), timeout)
    defer cancel()
    
    cmd := exec.CommandContext(ctx, name, args...)
    
    var stdout, stderr bytes.Buffer
    cmd.Stdout = &stdout
    cmd.Stderr = &stderr
    
    err := cmd.Run()
    
    result := CommandResult{
        Stdout:   stdout.String(),
        Stderr:   stderr.String(),
        Duration: time.Since(start),
        Err:      err,
    }
    
    if cmd.ProcessState != nil {
        result.ExitCode = cmd.ProcessState.ExitCode()
    }
    
    if ctx.Err() == context.DeadlineExceeded {
        result.Err = fmt.Errorf("command timed out after %v", timeout)
    }
    
    return result
}

// RunCommandAsync 异步执行命令
func RunCommandAsync(name string, args ...string) (<-chan CommandResult, context.CancelFunc) {
    ctx, cancel := context.WithCancel(context.Background())
    resultChan := make(chan CommandResult, 1)
    
    go func() {
        defer close(resultChan)
        
        cmd := exec.CommandContext(ctx, name, args...)
        var stdout, stderr bytes.Buffer
        cmd.Stdout = &stdout
        cmd.Stderr = &stderr
        
        start := time.Now()
        err := cmd.Run()
        
        result := CommandResult{
            Stdout:   stdout.String(),
            Stderr:   stderr.String(),
            Duration: time.Since(start),
            Err:      err,
        }
        
        if cmd.ProcessState != nil {
            result.ExitCode = cmd.ProcessState.ExitCode()
        }
        
        select {
        case resultChan <- result:
        case <-ctx.Done():
        }
    }()
    
    return resultChan, cancel
}

func main() {
    // 同步执行
    result := RunCommand(5*time.Second, "ls", "-la")
    fmt.Println("=== 同步执行 ===")
    fmt.Println("Stdout:", result.Stdout)
    fmt.Println("Duration:", result.Duration)
    fmt.Println("ExitCode:", result.ExitCode)
    
    // 异步执行
    fmt.Println("\n=== 异步执行 ===")
    resultChan, cancel := RunCommandAsync("sleep", "2")
    defer cancel()
    
    select {
    case result := <-resultChan:
        fmt.Println("命令完成，耗时:", result.Duration)
    case <-time.After(1 * time.Second):
        fmt.Println("不等了，取消命令")
        cancel()
    }
}
```

## 6. 僵尸进程和孤儿进程 ⭐⭐⭐

### 6.1 僵尸进程（Zombie Process）

**什么是僵尸进程？**
```
子进程终止后，父进程没有调用 wait() 回收，子进程就变成僵尸进程。

父进程                子进程
   │                    │
   │ fork()             │
   │ ──────────────────→│
   │                    │
   │                    │ 执行任务
   │                    │ ...
   │                    │ exit()
   │                    ↓
   │              ┌──────────┐
   │              │  僵尸    │ ← 进程已终止
   │              │ (Zombie) │   但 PCB 还在
   │              │  状态 Z  │   等待父进程回收
   │              └──────────┘
   │                    │
   │ wait()             │
   │ ──────────────────→│
   │                    ↓
   │                  彻底消失
```

**僵尸进程的危害：**
- 占用 PID（系统 PID 数量有限）
- 占用内核资源（PCB）
- 大量僵尸进程会导致无法创建新进程

**查看僵尸进程：**
```bash
# 查看僵尸进程
ps aux | grep Z

# 或者
ps aux | awk '$8=="Z" {print}'

# 统计僵尸进程数量
ps aux | awk '$8=="Z"' | wc -l
```

### 6.2 Go 中避免僵尸进程

```go

    "fmt"
    "os"
    "os/exec"
    "time"

    cmd := exec.Command("sleep", "1")
    cmd.Start()
    fmt.Println("子进程 PID:", cmd.Process.Pid)
    // 没有调用 Wait()，子进程结束后会变成僵尸
    
    time.Sleep(3 * time.Second)
    fmt.Println("检查僵尸进程: ps aux | grep", cmd.Process.Pid)
    
    // ========== 正确示例 1：同步等待 ==========
    fmt.Println("\n=== 正确示例 1：同步等待 ===")
    cmd = exec.Command("sleep", "1")
    cmd.Start()
    fmt.Println("子进程 PID:", cmd.Process.Pid)
    cmd.Wait() // 等待子进程结束并回收
    fmt.Println("子进程已回收")
    
    // ========== 正确示例 2：异步等待 ==========
    fmt.Println("\n=== 正确示例 2：异步等待 ===")
    cmd = exec.Command("sleep", "2")
    cmd.Start()
    fmt.Println("子进程 PID:", cmd.Process.Pid)
    
    go func() {
        cmd.Wait()
        fmt.Println("子进程已回收（异步）")
    }()
    
    fmt.Println("父进程继续执行其他任务...")
    time.Sleep(3 * time.Second)
    
    // ========== 正确示例 3：使用 Run() ==========
    fmt.Println("\n=== 正确示例 3：使用 Run() ===")
    cmd = exec.Command("echo", "hello")
    cmd.Run() // Run = Start + Wait
```

### 6.3 孤儿进程（Orphan Process）

**什么是孤儿进程？**
```
父进程先于子进程终止，子进程就变成孤儿进程。

父进程                子进程
   │                    │
   │ fork()             │
   │ ──────────────────→│
   │                    │
   │ exit()             │ 执行任务
   ↓                    │ ...
  消失                  │
                        │
init (PID=1)           │
   │                    │
   │ 收养               │
   │ ←──────────────────│
   │                    │
   │                    │ exit()
   │ wait()             │
   │ ──────────────────→│
   │                    ↓
   │                  彻底消失
```

**孤儿进程的特点：**
- 会被 init 进程（PID=1）收养
- init 进程会自动调用 wait() 回收
- 不会造成资源泄漏
- 是正常的，不需要担心

### 6.4 实际案例：守护进程

```go
package main

import (
    "fmt"
    "os"
    "os/exec"
    "syscall"
)

// 创建守护进程（daemon）
func main() {
    // 第一次 fork
    if os.Getppid() != 1 {
        // 不是由 init 启动的，需要 fork
        cmd := exec.Command(os.Args[0], os.Args[1:]...)
        cmd.Stdin = nil
        cmd.Stdout = nil
        cmd.Stderr = nil
        cmd.SysProcAttr = &syscall.SysProcAttr{
            Setsid: true, // 创建新会话
        }
        
        err := cmd.Start()
        if err != nil {
            fmt.Println("启动守护进程失败:", err)
            os.Exit(1)
        }
        
        fmt.Println("守护进程已启动，PID:", cmd.Process.Pid)
        os.Exit(0)
    }
    
    // 守护进程的主逻辑
    fmt.Println("守护进程运行中，PID:", os.Getpid())
    
    // 这里执行守护进程的任务
    select {} // 永久运行
}
```

### 🎯 练习题 6.1：僵尸进程

**问题：** 
1. 如何产生一个僵尸进程？
2. 如何清理僵尸进程？
3. 为什么 Go 程序中使用 `cmd.Start()` 后必须调用 `cmd.Wait()`？

<details>
<summary>点击查看答案</summary>

**1. 产生僵尸进程：**
```go
package main

import (
    "fmt"
    "os/exec"
    "time"
)

func main() {
    cmd := exec.Command("sleep", "1")
    cmd.Start()
    fmt.Println("子进程 PID:", cmd.Process.Pid)
    
    // 不调用 Wait()，子进程结束后变成僵尸
    time.Sleep(10 * time.Second)
}
```

**2. 清理僵尸进程：**
- 方法 1：让父进程调用 `Wait()`
- 方法 2：杀死父进程，让 init 收养并回收
- 方法 3：重启系统（最后手段）

```bash
# 找到僵尸进程的父进程
ps -ef | grep Z
# 杀死父进程
kill -9 <PPID>
```

**3. 为什么必须调用 Wait()：**
- `Start()` 只是启动子进程，不等待结束
- 子进程结束后，内核保留其退出状态等信息
- 必须调用 `Wait()` 读取这些信息，内核才会释放资源
- 不调用 `Wait()` 会导致僵尸进程
</details>

### 7.2 查看 Go 程序的线程

```go
package main

import (
    "fmt"
    "os"
    "runtime"
    "time"
)

func main() {
    fmt.Println("PID:", os.Getpid())
    fmt.Println("初始 GOMAXPROCS:", runtime.GOMAXPROCS(0))
    
    // 设置 P 的数量
    runtime.GOMAXPROCS(4)
    
    // 启动多个 goroutine
    for i := 0; i < 100; i++ {
        go func(id int) {
            for {
                // CPU 密集型任务
                sum := 0
                for j := 0; j < 1000000; j++ {
                    sum += j
                }
                time.Sleep(time.Millisecond)
            }
        }(i)
    }
    
    fmt.Println("Goroutines:", runtime.NumGoroutine())
    fmt.Println("\n查看线程数:")
    fmt.Println("  cat /proc", os.Getpid(), "/status | grep Threads")
    fmt.Println("  或")
    fmt.Println("  ps -T -p", os.Getpid())
    
    select {}
}
```

```bash
# 查看所有线程
```

## 8. 本章小结

### 核心概念

| 概念 | 说明 |
|------|------|
| 进程 | 程序的一次执行实例 |
| PCB | 进程控制块，进程的"身份证" |
| 进程状态 | 新建、就绪、运行、阻塞、终止 |
| fork() | 创建子进程 |
| 僵尸进程 | 子进程终止但未被回收 |
| 孤儿进程 | 父进程先终止，被 init 收养 |

### Go 相关

| 操作 | Go 实现 |
|------|---------|
| 获取 PID | `os.Getpid()` |
| 获取 PPID | `os.Getppid()` |
| 创建子进程 | `exec.Command()` |
| 等待子进程 | `cmd.Wait()` |
| 捕获信号 | `signal.Notify()` |
| 内存统计 | `runtime.MemStats` |

### 面试重点

1. **进程和线程的区别？**
   - 进程是资源分配的基本单位
   - 线程是 CPU 调度的基本单位
   - 进程有独立地址空间，线程共享进程地址空间

2. **僵尸进程和孤儿进程的区别？**
   - 僵尸：子进程终止，父进程未回收
   - 孤儿：父进程终止，子进程被 init 收养

3. **如何避免僵尸进程？**
   - 父进程调用 wait()/waitpid()
   - 使用信号处理 SIGCHLD
   - 两次 fork
