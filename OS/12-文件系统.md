## 2. inode ⭐⭐⭐

### 2.1 什么是 inode？

```
inode = 文件的元数据（不包含文件名！）

┌─────────────────────────────────────────────────────┐
│                     inode                            │
├─────────────────────────────────────────────────────┤
│  inode 号：12345                                    │
│  文件类型：普通文件 / 目录 / 链接 / ...             │
│  权限：rw-r--r-- (644)                              │
│  所有者 UID：1000                                   │
│  所属组 GID：1000                                   │
│  文件大小：1024 bytes                               │
│  链接数：1                                          │
│  时间戳：                                           │
│    - atime（最后访问时间）                          │
│    - mtime（最后修改时间）                          │
│    - ctime（状态改变时间）                          │
│  数据块指针：                                       │
│    - 直接指针 × 12                                  │
│    - 一级间接指针                                   │
│    - 二级间接指针                                   │
│    - 三级间接指针                                   │
└─────────────────────────────────────────────────────┘

注意：文件名不在 inode 中，而是在目录中！
```

### 2.3 Go 中获取文件信息

```go
package main

import (
    "fmt"
    "os"
    "syscall"
    "time"
)

func main() {
    fileInfo, err := os.Stat("test.txt")
    if err != nil {
        panic(err)
    }
    
    fmt.Println("=== 基本信息 ===")
    fmt.Println("文件名:", fileInfo.Name())
    fmt.Println("大小:", fileInfo.Size(), "bytes")
    fmt.Println("权限:", fileInfo.Mode())
    fmt.Println("修改时间:", fileInfo.ModTime().Format(time.RFC3339))
    fmt.Println("是否目录:", fileInfo.IsDir())
    
    // 获取系统特定信息（Linux）
    stat := fileInfo.Sys().(*syscall.Stat_t)
    
    fmt.Println("\n=== 系统信息 ===")
    fmt.Println("inode:", stat.Ino)
    fmt.Println("设备:", stat.Dev)
    fmt.Println("链接数:", stat.Nlink)
    fmt.Println("UID:", stat.Uid)
    fmt.Println("GID:", stat.Gid)
    fmt.Println("块数:", stat.Blocks)
}
```

### 📝 练习题 2.1

**问题**：为什么 inode 中不存储文件名？

<details>
<summary>点击查看答案</summary>

**答案**：

1. **支持硬链接**：
   - 一个 inode 可以有多个文件名
   - 如果文件名在 inode 中，硬链接就无法实现

2. **目录结构**：
   - 文件名是目录的一部分
   - 目录 = 文件名到 inode 的映射表
   - 这样设计更灵活

3. **重命名效率**：
   - 重命名只需修改目录项
   - 不需要修改 inode
   - 效率更高

</details>

## 3. 目录结构 ⭐⭐

### 3.1 目录是什么？

```
目录 = 文件名到 inode 的映射表

┌─────────────────────────────────────────────────────┐
│                    目录文件                          │
├──────────────────┬──────────────────────────────────┤
│     文件名       │          inode 号                │
├──────────────────┼──────────────────────────────────┤
│       .          │           100  (当前目录)        │
│       ..         │           50   (父目录)          │
│     file1.txt    │           101                    │
│     file2.txt    │           102                    │
│     subdir       │           103                    │
└──────────────────┴──────────────────────────────────┘
```

### 3.2 硬链接和软链接

```
硬链接：多个文件名指向同一个 inode

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  file1.txt  │────→│   inode     │←────│  file2.txt  │
│  (目录项)   │     │   12345     │     │  (目录项)   │
└─────────────┘     └──────┬──────┘     └─────────────┘
                           │
                           ↓
                    ┌─────────────┐
                    │   数据块    │
                    └─────────────┘

特点：
- 共享同一个 inode
- 删除一个不影响另一个
- 不能跨文件系统
- 不能链接目录

软链接（符号链接）：指向另一个文件名

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  link.txt   │────→│   inode     │────→│ "file1.txt" │
│  (目录项)   │     │   12346     │     │  (路径字符串)│
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                                               ↓
                                        ┌─────────────┐
                                        │  file1.txt  │
                                        │  (原文件)   │
                                        └─────────────┘

特点：
- 有自己的 inode
- 存储的是路径字符串
- 可以跨文件系统
- 可以链接目录
- 原文件删除后失效
```

### 3.3 Go 中创建链接

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    // 创建测试文件
    os.WriteFile("original.txt", []byte("Hello"), 0644)
    
    // 创建硬链接
    err := os.Link("original.txt", "hardlink.txt")
    if err != nil {
        fmt.Println("硬链接错误:", err)
    }
    
    // 创建软链接
    err = os.Symlink("original.txt", "softlink.txt")
    if err != nil {
        fmt.Println("软链接错误:", err)
    }
    
    // 读取软链接目标
    target, _ := os.Readlink("softlink.txt")
    fmt.Println("软链接目标:", target)
    
    // 查看链接数
    info, _ := os.Stat("original.txt")
    stat := info.Sys().(*syscall.Stat_t)
    fmt.Println("链接数:", stat.Nlink)  // 2（原文件 + 硬链接）
    
    // 清理
    os.Remove("original.txt")
    os.Remove("hardlink.txt")
    os.Remove("softlink.txt")
}
```

### 📝 练习题 3.1

**问题**：硬链接和软链接的区别？什么时候用哪个？

<details>
<summary>点击查看答案</summary>

| 特性 | 硬链接 | 软链接 |
|------|--------|--------|
| inode | 共享 | 独立 |
| 跨文件系统 | ❌ | ✅ |
| 链接目录 | ❌ | ✅ |
| 原文件删除 | 不影响 | 失效 |
| 存储内容 | 无 | 路径字符串 |

**使用场景**：
- **硬链接**：备份、节省空间、同一文件系统内
- **软链接**：跨文件系统、链接目录、配置文件

</details>

## 4. 文件描述符 ⭐⭐⭐

### 4.1 什么是文件描述符？

```
文件描述符（fd）= 进程打开文件的索引

┌─────────────────────────────────────────────────────┐
│                      进程                            │
│  ┌─────────────────────────────────────────────┐    │
│  │              文件描述符表                    │    │
│  ├─────┬───────────────────────────────────────┤    │
│  │ fd  │              指向                     │    │
│  ├─────┼───────────────────────────────────────┤    │
│  │  0  │  标准输入（stdin）                    │    │
│  │  1  │  标准输出（stdout）                   │    │
│  │  2  │  标准错误（stderr）                   │    │
│  │  3  │  打开的文件 A                         │    │
│  │  4  │  打开的文件 B                         │    │
│  │ ... │  ...                                  │    │
│  └─────┴───────────────────────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

### 4.2 文件描述符限制

```bash
# 查看当前限制
$ ulimit -n
1024

# 查看系统最大限制
$ cat /proc/sys/fs/file-max
100000

# 临时修改（当前 shell）
$ ulimit -n 65535

# 永久修改
# 编辑 /etc/security/limits.conf
# * soft nofile 65535
# * hard nofile 65535
```

### 4.3 Go 中的文件描述符

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    // 标准输入输出的 fd
    fmt.Println("stdin fd:", os.Stdin.Fd())   // 0
    fmt.Println("stdout fd:", os.Stdout.Fd()) // 1
    fmt.Println("stderr fd:", os.Stderr.Fd()) // 2
    
    // 打开文件
    file, _ := os.Open("test.txt")
    defer file.Close()
    
    // 获取文件描述符
    fd := file.Fd()
    fmt.Println("文件 fd:", fd)  // 通常是 3
}
```

## 5. Go 文件操作 ⭐⭐⭐

### 5.1 基本操作

```go
package main

import (
    "fmt"
    "io"
    "os"
)

func main() {
    // 创建文件
    file, _ := os.Create("test.txt")
    
    // 写入
    file.WriteString("Hello, World!\n")
    file.Write([]byte("Line 2\n"))
    
    // 移动偏移量到开头
    file.Seek(0, io.SeekStart)
    
    // 读取
    buf := make([]byte, 100)
    n, _ := file.Read(buf)
    fmt.Println("读取:", string(buf[:n]))
    
    file.Close()
    
    // 读取整个文件
    data, _ := os.ReadFile("test.txt")
    fmt.Println("全部内容:", string(data))
    
    // 写入整个文件
    os.WriteFile("output.txt", []byte("New content"), 0644)
    
    // 删除文件
    os.Remove("test.txt")
    os.Remove("output.txt")
}
```

### 5.2 目录操作

```go
package main

import (
    "fmt"
    "os"
    "path/filepath"
)

func main() {
    // 创建目录
    os.Mkdir("mydir", 0755)
    
    // 创建多级目录
    os.MkdirAll("a/b/c", 0755)
    
    // 读取目录
    entries, _ := os.ReadDir(".")
    for _, entry := range entries {
        info, _ := entry.Info()
        fmt.Printf("%s\t%d\t%s\n", 
            info.Mode(), info.Size(), entry.Name())
    }
    
    // 遍历目录（递归）
    filepath.WalkDir(".", func(path string, d os.DirEntry, err error) error {
        if err != nil {
            return err
        }
        fmt.Println(path)
        return nil
    })
    
    // 删除目录
    os.Remove("mydir")
    os.RemoveAll("a")
}
```

## 6. 本章小结

### 核心概念

| 概念 | 说明 |
|------|------|
| inode | 文件元数据，不包含文件名 |
| 目录 | 文件名到 inode 的映射 |
| 硬链接 | 多个文件名指向同一 inode |
| 软链接 | 指向另一个文件名 |
| 文件描述符 | 进程打开文件的索引 |

### Go 文件操作

| 操作 | 函数 |
|------|------|
| 打开 | `os.Open`, `os.Create`, `os.OpenFile` |
| 读写 | `file.Read`, `file.Write`, `io.Copy` |
| 读取全部 | `os.ReadFile`, `io.ReadAll` |
| 写入全部 | `os.WriteFile` |
| 目录 | `os.ReadDir`, `filepath.WalkDir` |
