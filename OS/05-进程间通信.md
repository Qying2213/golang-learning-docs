# 进程间通信（IPC）

> **重要程度：⭐⭐⭐ 必须掌握**  
> 理解 IPC 机制，才能理解微服务、消息队列等架构

## 1. 为什么需要进程间通信？

### 1.1 场景举例

```
场景1：微服务架构
┌─────────┐     ┌─────────┐     ┌─────────┐
│ 用户服务 │ ←→ │ 订单服务 │ ←→ │ 支付服务 │
└─────────┘     └─────────┘     └─────────┘
    ↑               ↑               ↑
    └───────────────┴───────────────┘
              需要通信！

场景2：生产者-消费者
┌─────────┐           ┌─────────┐
│ 生产者   │ ──数据──→ │ 消费者   │
│ 进程     │           │ 进程     │
└─────────┘           └─────────┘

场景3：父子进程
┌─────────┐
│ 父进程   │
└────┬────┘
     │ fork
     ↓
┌─────────┐
│ 子进程   │
└─────────┘
```

### 1.2 IPC 方式概览

| 方式 | 特点 | 适用场景 |
|------|------|---------|
| 管道 | 单向、亲缘进程 | 父子进程通信 |
| 命名管道 | 单向、任意进程 | 简单的进程通信 |
| 共享内存 | 最快、需要同步 | 大量数据传输 |
| 消息队列 | 异步、有格式 | 解耦的消息传递 |
| 信号 | 异步通知 | 进程控制 |
| Socket | 网络通信 | 跨机器通信 |

## 2. 管道（Pipe）⭐⭐⭐

### 2.1 匿名管道

```
┌─────────┐    写入    ┌─────────┐    读取    ┌─────────┐
│ 父进程   │ ────────→ │  管道    │ ────────→ │ 子进程   │
│ (写端)   │           │ (缓冲区) │           │ (读端)   │
└─────────┘           └─────────┘           └─────────┘
```

**特点：**
- 单向通信（半双工）
- 只能用于有亲缘关系的进程
- 数据先进先出（FIFO）
- 有容量限制（通常 64KB）

### 2.2 Go 中使用管道

```go
package main

import (
    "fmt"
    "io"
    "os/exec"
)

func main() {
    // 创建命令
    cmd := exec.Command("cat")
    
    // 获取输入管道
    stdin, _ := cmd.StdinPipe()
    
    // 获取输出管道
    stdout, _ := cmd.StdoutPipe()
    
    // 启动命令
    cmd.Start()
    
    // 写入数据
    io.WriteString(stdin, "Hello, Pipe!\n")
    stdin.Close()
    
    // 读取输出
    output, _ := io.ReadAll(stdout)
    fmt.Println("输出:", string(output))
    
    cmd.Wait()
}
```

### 2.3 管道连接多个命令

```go
package main

import (
    "fmt"
    "os/exec"
)

func main() {
    // 模拟: cat file.txt | grep "error" | wc -l
    
    // 第一个命令
    cat := exec.Command("echo", "error\ninfo\nerror\nwarn")
    
    // 第二个命令
    grep := exec.Command("grep", "error")
    
    // 第三个命令
    wc := exec.Command("wc", "-l")
    
    // 连接管道
    grep.Stdin, _ = cat.StdoutPipe()
    wc.Stdin, _ = grep.StdoutPipe()
    
    // 获取最终输出
    output, _ := wc.StdoutPipe()
    
    // 按顺序启动
    cat.Start()
    grep.Start()
    wc.Start()
    
    // 读取结果
    result := make([]byte, 100)
    n, _ := output.Read(result)
    fmt.Println("包含 error 的行数:", string(result[:n]))
    
    // 等待完成
    cat.Wait()
    grep.Wait()
    wc.Wait()
}
```

### 2.4 命名管道（FIFO）

```bash
# 创建命名管道
mkfifo /tmp/myfifo

# 终端1：写入
echo "Hello" > /tmp/myfifo

# 终端2：读取
cat /tmp/myfifo
```

```go
package main

import (
    "fmt"
    "os"
    "syscall"
)

func main() {
    fifoPath := "/tmp/myfifo"
    
    // 创建命名管道
    syscall.Mkfifo(fifoPath, 0666)
    
    // 写入进程
    go func() {
        file, _ := os.OpenFile(fifoPath, os.O_WRONLY, 0666)
        defer file.Close()
        file.WriteString("Hello from writer!\n")
    }()
    
    // 读取进程
    file, _ := os.OpenFile(fifoPath, os.O_RDONLY, 0666)
    defer file.Close()
    
    buf := make([]byte, 100)
    n, _ := file.Read(buf)
    fmt.Println("读取:", string(buf[:n]))
}
```

## 3. 共享内存 ⭐⭐

### 3.1 原理

```
┌─────────────────────────────────────────────────────┐
│                    物理内存                          │
│  ┌─────────────────────────────────────────────┐    │
│  │              共享内存区域                    │    │
│  └─────────────────────────────────────────────┘    │
│         ↑                           ↑               │
│         │ 映射                      │ 映射          │
│  ┌──────┴──────┐             ┌──────┴──────┐       │
│  │   进程 A    │             │   进程 B    │       │
│  │  虚拟地址   │             │  虚拟地址   │       │
│  └─────────────┘             └─────────────┘       │
└─────────────────────────────────────────────────────┘
```

**特点：**
- 最快的 IPC 方式（直接访问内存）
- 需要同步机制（如信号量）
- 数据持久化到进程退出

### 3.2 Go 中使用共享内存

```go
package main

import (
    "fmt"
    "os"
    "syscall"
    "unsafe"
)

func main() {
    // 创建共享内存文件
    file, err := os.OpenFile("/dev/shm/myshm", os.O_RDWR|os.O_CREATE, 0666)
    if err != nil {
        panic(err)
    }
    defer file.Close()
    
    // 设置大小
    size := 1024
    file.Truncate(int64(size))
    
    // 内存映射
    data, err := syscall.Mmap(
        int(file.Fd()),
        0,
        size,
        syscall.PROT_READ|syscall.PROT_WRITE,
        syscall.MAP_SHARED,
    )
    if err != nil {
        panic(err)
    }
    defer syscall.Munmap(data)
    
    // 写入数据
    message := "Hello, Shared Memory!"
    copy(data, message)
    
    // 读取数据
    fmt.Println("读取:", string(data[:len(message)]))
}
```

### 3.3 使用 mmap 文件

```go
package main

import (
    "fmt"
    "os"
    "syscall"
)

func main() {
    // 创建文件
    file, _ := os.Create("shared.dat")
    file.Truncate(4096)
    file.Close()
    
    // 进程 A：写入
    go func() {
        f, _ := os.OpenFile("shared.dat", os.O_RDWR, 0666)
        defer f.Close()
        
        data, _ := syscall.Mmap(int(f.Fd()), 0, 4096,
            syscall.PROT_READ|syscall.PROT_WRITE, syscall.MAP_SHARED)
        defer syscall.Munmap(data)
        
        copy(data, "Message from Process A")
        fmt.Println("A: 写入完成")
    }()
    
    // 进程 B：读取
    go func() {
        f, _ := os.OpenFile("shared.dat", os.O_RDONLY, 0666)
        defer f.Close()
        
        data, _ := syscall.Mmap(int(f.Fd()), 0, 4096,
            syscall.PROT_READ, syscall.MAP_SHARED)
        defer syscall.Munmap(data)
        
        fmt.Println("B: 读取:", string(data[:22]))
    }()
    
    select {}
}
```

## 4. 信号（Signal）⭐⭐⭐

### 4.1 常用信号

| 信号 | 数字 | 说明 | 默认行为 |
|------|------|------|---------|
| SIGINT | 2 | Ctrl+C | 终止 |
| SIGTERM | 15 | kill 命令 | 终止 |
| SIGKILL | 9 | 强制终止 | 终止（不可捕获）|
| SIGHUP | 1 | 终端断开 | 终止 |
| SIGUSR1 | 10 | 用户自定义 | 终止 |
| SIGUSR2 | 12 | 用户自定义 | 终止 |
| SIGCHLD | 17 | 子进程状态改变 | 忽略 |

### 4.2 Go 中处理信号

```go
package main

import (
    "fmt"
    "os"
    "os/signal"
    "syscall"
)

func main() {
    // 创建信号通道
    sigChan := make(chan os.Signal, 1)
    
    // 注册要捕获的信号
    signal.Notify(sigChan,
        syscall.SIGINT,  // Ctrl+C
        syscall.SIGTERM, // kill
        syscall.SIGHUP,  // 终端断开
        syscall.SIGUSR1, // 用户自定义
    )
    
    fmt.Println("程序运行中，PID:", os.Getpid())
    fmt.Println("发送信号: kill -USR1", os.Getpid())
    
    for sig := range sigChan {
        switch sig {
        case syscall.SIGINT, syscall.SIGTERM:
            fmt.Println("\n收到终止信号，退出...")
            return
        case syscall.SIGHUP:
            fmt.Println("收到 SIGHUP，重新加载配置...")
            // 重新加载配置
        case syscall.SIGUSR1:
            fmt.Println("收到 SIGUSR1，执行自定义操作...")
            // 自定义操作
        }
    }
}
```

### 4.3 优雅退出

```go
package main

import (
    "context"
    "fmt"
    "net/http"
    "os"
    "os/signal"
    "syscall"
    "time"
)

func main() {
    server := &http.Server{Addr: ":8080"}
    
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        time.Sleep(2 * time.Second) // 模拟耗时操作
        fmt.Fprintln(w, "Hello!")
    })
    
    // 启动服务器
    go func() {
        fmt.Println("服务器启动在 :8080")
        server.ListenAndServe()
    }()
    
    // 等待信号
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
    <-quit
    
    fmt.Println("\n正在关闭服务器...")
    
    // 优雅关闭，等待 10 秒
    ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()
    
    if err := server.Shutdown(ctx); err != nil {
        fmt.Println("关闭错误:", err)
    }
    
    fmt.Println("服务器已关闭")
}
```

### 4.4 发送信号

```go
package main

import (
    "fmt"
    "os"
    "os/exec"
    "syscall"
    "time"
)

func main() {
    // 启动子进程
    cmd := exec.Command("sleep", "100")
    cmd.Start()
    
    fmt.Println("子进程 PID:", cmd.Process.Pid)
    
    time.Sleep(2 * time.Second)
    
    // 发送信号
    cmd.Process.Signal(syscall.SIGTERM)
    fmt.Println("已发送 SIGTERM")
    
    // 或者使用 os.Process
    process, _ := os.FindProcess(cmd.Process.Pid)
    process.Signal(syscall.SIGKILL)
    
    cmd.Wait()
}
```

## 5. Socket 通信 ⭐⭐⭐

### 5.1 Unix Domain Socket

```
┌─────────┐                      ┌─────────┐
│ 进程 A  │ ←── Unix Socket ──→ │ 进程 B  │
│ (客户端)│    /tmp/app.sock    │ (服务端)│
└─────────┘                      └─────────┘
```

**服务端：**
```go
package main

import (
    "fmt"
    "net"
    "os"
)

func main() {
    socketPath := "/tmp/myapp.sock"
    
    // 删除旧的 socket 文件
    os.Remove(socketPath)
    
    // 监听
    listener, err := net.Listen("unix", socketPath)
    if err != nil {
        panic(err)
    }
    defer listener.Close()
    
    fmt.Println("服务端启动，监听:", socketPath)
    
    for {
        conn, err := listener.Accept()
        if err != nil {
            continue
        }
        
        go handleConnection(conn)
    }
}

func handleConnection(conn net.Conn) {
    defer conn.Close()
    
    buf := make([]byte, 1024)
    n, _ := conn.Read(buf)
    
    fmt.Println("收到:", string(buf[:n]))
    conn.Write([]byte("Hello from server!"))
}
```

**客户端：**
```go
package main

import (
    "fmt"
    "net"
)

func main() {
    conn, err := net.Dial("unix", "/tmp/myapp.sock")
    if err != nil {
        panic(err)
    }
    defer conn.Close()
    
    // 发送
    conn.Write([]byte("Hello from client!"))
    
    // 接收
    buf := make([]byte, 1024)
    n, _ := conn.Read(buf)
    fmt.Println("收到:", string(buf[:n]))
}
```

### 5.2 TCP Socket

```go
// 服务端
package main

import (
    "bufio"
    "fmt"
    "net"
)

func main() {
    listener, _ := net.Listen("tcp", ":8080")
    defer listener.Close()
    
    fmt.Println("TCP 服务端启动在 :8080")
    
    for {
        conn, _ := listener.Accept()
        go func(c net.Conn) {
            defer c.Close()
            
            reader := bufio.NewReader(c)
            for {
                line, err := reader.ReadString('\n')
                if err != nil {
                    return
                }
                fmt.Print("收到:", line)
                c.Write([]byte("ACK: " + line))
            }
        }(conn)
    }
}
```

```go
// 客户端
package main

import (
    "bufio"
    "fmt"
    "net"
)

func main() {
    conn, _ := net.Dial("tcp", "localhost:8080")
    defer conn.Close()
    
    // 发送
    fmt.Fprintln(conn, "Hello, TCP!")
    
    // 接收
    reader := bufio.NewReader(conn)
    response, _ := reader.ReadString('\n')
    fmt.Println("响应:", response)
}
```

## 6. 消息队列 ⭐⭐

### 6.1 使用 Channel 模拟

```go
package main

import (
    "fmt"
    "time"
)

type Message struct {
    ID      int
    Content string
}

func main() {
    // 消息队列
    queue := make(chan Message, 100)
    
    // 生产者
    go func() {
        for i := 1; i <= 10; i++ {
            msg := Message{ID: i, Content: fmt.Sprintf("消息 %d", i)}
            queue <- msg
            fmt.Println("生产:", msg)
            time.Sleep(100 * time.Millisecond)
        }
        close(queue)
    }()
    
    // 消费者
    for msg := range queue {
        fmt.Println("消费:", msg)
        time.Sleep(200 * time.Millisecond)
    }
}
```

### 6.2 实际应用：Redis 消息队列

```go
package main

import (
    "context"
    "fmt"
    "github.com/redis/go-redis/v9"
    "time"
)

func main() {
    ctx := context.Background()
    
    rdb := redis.NewClient(&redis.Options{
        Addr: "localhost:6379",
    })
    
    // 生产者
    go func() {
        for i := 1; i <= 10; i++ {
            rdb.LPush(ctx, "myqueue", fmt.Sprintf("message-%d", i))
            fmt.Println("发送消息:", i)
            time.Sleep(time.Second)
        }
    }()
    
    // 消费者
    for {
        result, err := rdb.BRPop(ctx, 0, "myqueue").Result()
        if err != nil {
            continue
        }
        fmt.Println("收到消息:", result[1])
    }
}
```

## 7. IPC 方式对比 ⭐⭐⭐

| 方式 | 速度 | 复杂度 | 适用场景 |
|------|------|--------|---------|
| 管道 | 中 | 低 | 父子进程、简单数据流 |
| 命名管道 | 中 | 低 | 任意进程、简单通信 |
| 共享内存 | 快 | 高 | 大量数据、高性能 |
| 信号 | 快 | 低 | 进程控制、通知 |
| Unix Socket | 中 | 中 | 本机进程通信 |
| TCP Socket | 慢 | 中 | 跨机器通信 |
| 消息队列 | 中 | 中 | 异步解耦 |

**选择建议：**
- 父子进程简单通信 → 管道
- 本机进程通信 → Unix Socket
- 跨机器通信 → TCP Socket
- 大量数据传输 → 共享内存
- 进程控制 → 信号
- 异步解耦 → 消息队列（Redis/RabbitMQ）

实现一个程序，使用管道执行 `ls -la | grep ".go" | wc -l`

```go
package main

import (
    "fmt"
    "io"
    "os/exec"
)

func main() {
    ls := exec.Command("ls", "-la")
    grep := exec.Command("grep", ".go")
    wc := exec.Command("wc", "-l")
    
    // 连接管道
    lsOut, _ := ls.StdoutPipe()
    grep.Stdin = lsOut
    
    grepOut, _ := grep.StdoutPipe()
    wc.Stdin = grepOut
    
    wcOut, _ := wc.StdoutPipe()
    
    // 启动
    ls.Start()
    grep.Start()
    wc.Start()
    
    // 读取结果
    result, _ := io.ReadAll(wcOut)
    
    // 等待完成
    ls.Wait()
    grep.Wait()
    wc.Wait()
    
    fmt.Println("Go 文件数量:", string(result))
}
```

### 练习2：信号处理

实现一个程序：
- 收到 SIGUSR1 时打印当前时间
- 收到 SIGUSR2 时打印 goroutine 数量
- 收到 SIGINT 时优雅退出

<details>
<summary>点击查看答案</summary>

```go
package main

import (
    "fmt"
    "os"
    "os/signal"
    "runtime"
    "syscall"
    "time"
)

func main() {
    sigChan := make(chan os.Signal, 1)
    signal.Notify(sigChan, syscall.SIGUSR1, syscall.SIGUSR2, syscall.SIGINT)
    
    fmt.Println("PID:", os.Getpid())
    fmt.Println("测试命令:")
    fmt.Println("  kill -USR1", os.Getpid())
    fmt.Println("  kill -USR2", os.Getpid())
    fmt.Println("  kill -INT", os.Getpid())
    
    // 启动一些 goroutine
    for i := 0; i < 5; i++ {
        go func() { select {} }()
    }
    
    for sig := range sigChan {
        switch sig {
        case syscall.SIGUSR1:
            fmt.Println("当前时间:", time.Now().Format("2006-01-02 15:04:05"))
        case syscall.SIGUSR2:
            fmt.Println("Goroutine 数量:", runtime.NumGoroutine())
        case syscall.SIGINT:
            fmt.Println("\n正在退出...")
            return
        }
    }
}
```

## 9. 本章小结

**IPC 方式：**

| 方式 | Go 实现 |
|------|---------|
| 管道 | `exec.Command` + `StdinPipe/StdoutPipe` |
| 命名管道 | `syscall.Mkfifo` + `os.OpenFile` |
| 共享内存 | `syscall.Mmap` |
| 信号 | `signal.Notify` |
| Unix Socket | `net.Listen("unix", path)` |
| TCP Socket | `net.Listen("tcp", addr)` |

**最佳实践：**
- 优先使用 channel（goroutine 间通信）
- 本机进程通信用 Unix Socket
- 跨机器用 TCP/HTTP/gRPC
- 异步解耦用消息队列
