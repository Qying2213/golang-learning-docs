# 🎓 毕业设计冲刺指南：用 Claude 快速完成人才平台项目

## 📋 项目开发流程 + Claude 任务分配

### Phase 1：架构设计（1-2天）

#### 🔹 Prompt 模板：系统架构设计

```
我正在开发一个"智能人才运营平台"作为本科毕业设计，技术栈如下：
- 后端：Go + Gin 框架
- 数据库：MySQL + Redis
- 前端：Vue3 + TypeScript

请帮我设计完整的系统架构，包括：
1. 系统整体架构图（用文字描述，我可以据此画图）
2. 模块划分和职责说明
3. 数据库表设计（所有表的字段、类型、索引）
4. API 接口列表（RESTful 风格）
5. 目录结构建议

请尽可能详细和完整。
```

#### 🔹 Prompt 模板：数据库表设计

```
请帮我设计人才平台的数据库表，需要支持以下功能：
1. 用户注册登录（求职者/企业/管理员三种角色）
2. 求职者个人信息和简历管理
3. 企业信息和职位发布
4. 职位申请和状态追踪
5. 消息通知
6. 智能推荐记录

要求：
- 使用 MySQL
- 给出完整的建表 SQL
- 包含必要的索引
- 添加注释说明每个字段的用途
```

### Phase 2：后端开发（1-2周）

#### 🔹 Prompt 模板：生成完整的模块代码

```
请帮我用 Go + Gin 框架实现【用户模块】的完整代码，包括：

1. model/user.go - 用户模型定义
2. repository/user_repo.go - 数据库操作层
3. service/user_service.go - 业务逻辑层
4. handler/user_handler.go - API 处理器
5. router 注册代码

功能需求：
- 用户注册（手机号+密码+验证码）
- 用户登录（返回 JWT token）
- 获取用户信息
- 修改用户信息
- 修改密码

技术要求：
- 密码使用 bcrypt 加密
- 使用 JWT 进行身份验证
- 统一的响应格式
- 完整的错误处理
- 代码要有注释

请生成完整可运行的代码。
```

#### 🔹 各模块开发 Prompt 清单

| 模块         | Prompt 关键点                       |
| ------------ | ----------------------------------- |
| **用户模块** | 注册、登录、JWT、角色权限、个人信息 |
| **简历模块** | 简历创建、编辑、附件上传、简历解析  |
| **职位模块** | 职位发布、搜索、筛选、状态管理      |
| **申请模块** | 投递申请、状态流转、企业处理        |
| **推荐模块** | 基于标签匹配、基于历史行为          |
| **消息模块** | 站内信、通知推送、已读状态          |

#### 🔹 Prompt 模板：生成中间件

```
请帮我用 Go + Gin 实现以下中间件：

1. JWT 认证中间件
   - 从 Header 中提取 token
   - 验证 token 有效性
   - 将用户信息注入 context

2. 角色权限中间件
   - 支持配置允许的角色列表
   - 验证当前用户是否有权限

3. 请求日志中间件
   - 记录请求方法、路径、耗时
   - 记录响应状态码

4. 错误恢复中间件
   - 捕获 panic
   - 返回统一错误响应

请生成完整代码，包含使用示例。
```

## 🎯 每日开发 Checklist

```
□ 早上确定今天要完成的 2-3 个功能点
□ 让 Claude 生成代码框架和核心逻辑
□ 复制代码到项目中，调试运行
□ 遇到 bug 立即让 Claude 帮忙 debug
□ 功能完成后，让 Claude 生成对应的注释和文档
□ 晚上提交代码，记录当日进度
```

## ⚠️ 常见问题处理

### Q: Claude 生成的代码有 bug 怎么办？

直接把错误信息贴给它，说"这段代码运行报错了，请修复"

### Q: 代码太长，一次生成不完？

说"请继续"，或者分模块请求

### Q: 生成的代码风格不统一？

在 prompt 开头加上你的代码规范要求

---

> 💡 **核心建议**：不要试图一次让 Claude 生成整个项目，而是模块化、渐进式地开发。每完成一个模块就测试一个模块。
