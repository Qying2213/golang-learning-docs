## ⚡ 使用技巧：让每次对话产出最大化

### 技巧1：批量请求

```
❌ 低效：一次只问一个问题
✅ 高效：一次请求多个相关内容

示例："请帮我：
1. 设计用户模块的数据库表结构
2. 生成对应的 Go 结构体
3. 编写 CRUD 的 Repository 层代码
4. 编写对应的 Service 层代码
5. 生成 API Handler 代码"
```

### 技巧2：要求完整输出

```
❌ 低效："帮我写个用户登录功能"
✅ 高效："帮我用 Go + Gin 框架实现完整的用户登录功能，包括：
   - JWT token 生成和验证
   - 密码加密（bcrypt）
   - 登录 API handler
   - 中间件鉴权
   - 完整的错误处理
   - 单元测试代码
   请输出完整可运行的代码"
```

### 技巧3：让它生成文档

```
每完成一个功能，立即让 Claude 生成：
- API 文档（Swagger 格式）
- 使用说明
- 代码注释
- 测试用例
```

### 技巧4：保存所有输出

```
⚠️ 重要：Claude 的输出不会自动保存！

每次有价值的对话，立即：
1. 复制代码到本地文件
2. 复制文档到 Notion/Obsidian
3. 截图保存重要的思路分析
```

## ⚠️ 注意事项

1. **不要浪费在闲聊上** - 每次对话都要有明确产出
2. **及时保存** - Claude 对话可能会丢失，有价值的内容立即保存
3. **先规划再执行** - 每天开始前，列出要让 Claude 完成的任务清单
4. **复杂任务分解** - 大任务拆成小任务，每次专注一个模块
